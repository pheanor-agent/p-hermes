# Knowledge System 설계 철학: 왜 에이전트는 구조화된 기억이 필요한가

> **주제**: #knowledge-system #architecture #memory-management  
> **예상 읽기 시간**: 15분

---

## 문제: 에이전트의 기억은 어떻게 저장해야 하는가

AI 에이전트의 가장 큰 한계 중 하나는 **영속성**입니다.

대화 세션이 끝나면 모든 컨텍스트가 사라집니다. 사용자의 선호도, 프로젝트의 결정 사항, 학습한 내용 — 모두 초기화됩니다. 이 문제를 해결하기 위해 단순히 "모든 것을 저장하자"는 접근은 실용적이지 않습니다. 저장 공간은 유한하고, 검색은 빨라야 하며, 오래된 정보는 새로운 정보와 충돌할 수 있습니다.

Hermes Knowledge System은 이 문제를 **3-Tier 구조**로 해결합니다.

---

## 3-Tier Knowledge Architecture

```
Tier 1 (원시): 수집된 그대로의 데이터
Tier 2 (가공): 정제되고 구조화된 지식
Tier 3 (추론): 추론과 통찰이 포함된 고급 지식
```

### Tier 1: Raw — 수집

첫 번째 계층은 **수집된 그대로의 데이터**입니다. 뉴스 기사, 문서, 로그 등 필터링 없이 저장됩니다.

- 저장 위치: `~/.hermes/knowledge/raw/`
- 특징: 빠른 수집, 최소한의 가공
- 사용처: 검색 인덱싱, 사전 필터링

### Tier 2: Processed — 가공

두 번째 계층은 **정제되고 구조화된 지식**입니다. 중복 제거, 분류, 태깅, 요약을 거친 데이터입니다.

- 저장 위치: `~/.hermes/knowledge/wiki/`
- 특징: 구조화된 메타데이터, 교차 참조 가능
- 사용처: Knowledge 검색, Wiki 참조

### Tier 3: Reasoned — 추론

세 번째 계층은 **추론과 통찰이 포함된 지식**입니다. 여러 출처의 정보를 종합하고, 패턴을 발견하고, 결론을 도출한 결과물입니다.

- 저장 위치: `~/.hermes/knowledge/reasoned/`
- 특징: 높은 정보 밀도, 판단 포함
- 사용처: 의사 결정, 장기 전략

## 점수화 시스템 (Scoring)

모든 Knowledge 항목은 **점수(Score)**를 가집니다. 이 점수는 검색 순위와 중요도 결정에 사용됩니다.

| 요소 | 배점 | 설명 |
|:----:|:----:|------|
| 관련성 | 0-40 | 검색어와의 일치도 |
| 최신성 | 0-30 | 최근일수록 높은 점수 |
| 출처 신뢰도 | 0-20 | 원본 출처의 권위 |
| 사용 빈도 | 0-10 | 참조 횟수 |

---

## 동기화 파이프라인

Knowledge는 수집부터 검색까지 자동화된 파이프라인을 통해 처리됩니다.

```
수집 → 정제 → 분류 → 점수화 → 색인 → 검색
  │       │       │       │       │       │
  │       │       │       │       │       └─ FTS5 검색
  │       │       │       │       └─ 색인 등록
  │       │       │       └─ Score 계산
  │       │       └─ 도메인 분류
  │       └─ 중복 제거
  └─ 소스 수집
```

---

## 철학: 왜 이 구조인가

Knowledge System의 핵심 설계 원칙은 **"잊지 말되, 모든 것을 기억하지는 않는다"**입니다.

1. **계층화**: 모든 데이터를 같은 수준으로 저장하지 않습니다. 원시 데이터는 빠르게 쌓고, 중요한 지식만 정제합니다.

2. **점수화**: 모든 지식이 동등하지 않습니다. 점수 시스템을 통해 중요한 지식이 먼저 검색되도록 합니다.

3. **자동화**: 수동 분류는 확장되지 않습니다. 파이프라인이 자동으로 처리합니다.

4. **연결성**: Knowledge 항목은 서로 연결됩니다. "이 결정은 어떤 배경에서 나왔는가?"를 추적할 수 있습니다.

---

## 결론

Knowledge System은 단순한 "저장소"가 아닙니다. 에이전트가 시간이 지날수록 더 똑똑해질 수 있게 만드는 **지식 인프라**입니다. 3-Tier 구조는 수집 속도와 정보 품질 사이의 균형을 제공하고, 점수화 시스템은 검색 정확도를 보장합니다.
