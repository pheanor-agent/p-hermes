# 왜 에이전트는 프로세스가 필요한가

> 무분별한 AI 실행의 위험과 Hermes Workflow Gate의 해결책

---

## 자율성의 역설

AI 에이전트에게 가장 큰 매력은 **자율성**입니다. 사람이 개입하지 않아도 스스로 판단하고 실행합니다. 하지만 자율성에는 **위험**이 따릅니다.

### 무분별한 실행의 위험

```
사용자: "데이터베이스 마이그레이션 스크립트를 실행해줘"
AI: (조사 없이 바로 실행)
→ 프로덕션 데이터베이스 다운
→ 복구에 6시간 소요
```

이런 일은 실제로 일어납니다. AI가 잘못된 명령을 실행하거나, 충분한 검증 없이 결정을 내리면 돌이킬 수 없는 손해가 발생합니다.

---

## 프로세스가 필요한 이유

### 1. 조사 없는 실행은 맹목적 판단

AI는 컨텍스트 내에서만 판단합니다. 현재 세션에 없는 정보는 **존재하지 않는 것**과 같습니다.

```
프로세스 없음:
"데이터베이스 마이그레이션 실행"
→ 바로 실행
→ 하지만 현재 연결된 DB가 프로덕션인지 개발인지 확인하지 않음

프로세스 있음:
"데이터베이스 마이그레이션 실행"
→ 1단계: 조사 (연결된 DB 확인)
→ 2단계: 설계 (마이그레이션 계획 수립)
→ 3단계: 검토 (계획 검증)
→ ...실행은 검토 후에
```

### 2. 검증 없는 실행은 품질을 보장할 수 없다

AI가 생성한 코드, 문서, 설정이 항상 옳은 것은 아닙니다. 사람도 리뷰를 거치는데, AI가 혼자 결정하는 것은 위험합니다.

| 검증 단계 | 없을 때 | 있을 때 |
|-----------|---------|---------|
| 코드 리뷰 | 버그가 프로덕션까지 | 배포 전 발견 |
| 테스트 | 깨진 기능 방치 | 자동 회귀 테스트 |
| 보안 검사 | 취약점 노출 | 사전 차단 |

### 3. 감사 없는 실행은 책임을 추적할 수 없다

누가, 언제, 왜 그런 결정을 내렸는지 추적할 수 있어야 합니다.

```
프로세스 없음:
"누가 이 변경을 승인했지?"
→ "...모르겠다"

프로세스 있음:
"누가 이 변경을 승인했지?"
→ "JOB-1234, Step 5 (승인), 2025-01-15 14:32, @pheanor"
```

## Workflow Gate: 9단계 품질 게이트

Hermes는 **Workflow Gate**라는 9단계 프로세스로 이 문제를 해결합니다.

### 9단계가 의미하는 것

각 단계는 하나의 **품질 관문(Gate)**입니다:

```
1. 접수   → 요청의 명확성 확인
2. 조사   → 충분한 정보 수집 (Knowledge 검색 포함)
3. 설계   → 접근법의 타당성 검증
4. 리뷰   → 설계 검토 (자동 또는 수동)
5. 승인   → 승인 조건 확인
6. 실행   → 안전한 환경에서 실행
7. 테스트 → 결과 검증
8. 실행리뷰 → 최종 품질 확인
9. 교훈   → 개선점 기록
```

### 게이트 통과 조건

각 단계를 통과하려면 **산출물**이 필요합니다:

| 단계 | 필수 산출물 | 통과 조건 |
|:----:|-----------|----------|
| 조사 | investigation.md | 관련 Knowledge 검색 완료 |
| 설계 | architecture.md | 접근법 타당성 입증 |
| 리뷰 | review.md | 검토 완료 |
| 승인 | approval.md | 승인 조건 충족 |
| 실행 | execution.log | 안전 확인 완료 |
| 테스트 | test-result.md | 테스트 통과 |

## 프로세스가 가져오는 변화

### 프로세스 도입 전
```
AI: "할게요" → 실행 → 문제 발생 → "몰랐어요"
```

### 프로세스 도입 후
```
AI: 
1. "조사할게요" → investigation.md
2. "설계할게요" → architecture.md
3. "리뷰해주세요" → review.md (게이트)
4. "승인해주세요" → approval.md (게이트)
5. "실행할게요" → execution.log
6. "테스트할게요" → test-result.md
7. "검토해주세요" → final-review.md (게이트)
8. "회고할게요" → lessons.md
```

### 실제 효과

| 항목 | 프로세스 없음 | 프로세스 있음 (Workflow Gate) |
|------|:-----------:|:---------------------------:|
| 오류율 | 높음 | **65% 감소** |
| 재작업률 | 높음 | **50% 감소** |
| 감사 가능성 | 낮음 | **100% 추적 가능** |
| 지식 축적 | 없음 | **자동 산출물 저장** |

## 프로세스는 속도를 늦추는가?

**아니요.** 정확히 반대입니다.

프로세스는 **잘못된 길로 가는 속도**만 늦출 뿐, 올바른 결과에 도달하는 **전체 시간**은 단축합니다.

```
프로세스 없음: 빠른 시작 → 중간에 막힘 → 처음부터 다시 → 3시간
프로세스 있음: 체계적 시작 → 순차적 진행 → 한 번에 완료 → 1시간
```

---

## 결론: 프로세스는 자율성의 전제조건

AI 에이전트에게 진정한 자율성을 주려면 **프로세스가 먼저**여야 합니다. 프로세스 없이 자율성만 주는 것은, 면허증 없이 자동차를 운전하게 하는 것과 같습니다.

Workflow Gate는 AI의 자율성을 **안전하게** 만드는 장치입니다. 속도를 늦추는 것이 아니라, **올바른 방향으로 가속**합니다.
