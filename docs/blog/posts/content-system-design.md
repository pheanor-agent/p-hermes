# 콘텐츠 시스템 설계: AI의 표현력을 구조화하는 법

## 🌱 기본 개념: AI에게 '표현력'이란 무엇일까요?

AI에게 표현력이란 단순히 문장을 만드는 것이 아닙니다. **타겟 독자층에 맞춰 정보의 깊이를 조절하고, 어조를 일치시키며, 일관된 톤앤매너를 유지하는 것**입니다.

콘텐츠 시스템은 이러한 표현력을 자동화하는 파이프라인입니다.

---

## 🔍 문제 상황: "일관성 없이 반복되는 콘텐츠 품질 문제"

콘텐츠 시스템을 도입하기 전, 다음과 같은 문제가 있었습니다:

1. **타겟 독자층 무시** - 초급자도 전문가도 동일한 수준의 글
2. **어조 불일치** - 전문 용어와 구어체가 혼재
3. **부정 대조 패턴 남용** - "~가 아니라 ~이다" 구문 과다
4. **품질 검증 부재** - AI-isms, 중복 표현, 불필요한 수식어

### 해결책: 콘텐츠 시스템 (Content System)

콘텐츠 시스템은 **도메인 분류 → 엔진 처리 → 검증**의 3단계 파이프라인으로 이러한 문제를 해결합니다.

---

## 🏗️ 콘텐츠 시스템 아키텍처

```
요청 수신
    ↓
[도메인 분류] → D1:교육/설명, D2:내러티브, D3:시각, D4:프레젠테이션, D5:이미지
    ↓
[엔진 처리] → Tier Generator → Tone Adapter → Analogy Builder → Template Filler
    ↓
[검증 게이트] → L1:Schema, L2:Error, L3:Voice, L4:Domain, L5:Judge
    ↓
최종 출력
```

---

## 📋 도메인 분류

| 도메인 | 설명 | 하위 유형 |
|--------|------|-----------|
| **D1: 교육/설명** | README, Wiki, 블로그, 가이드 | 설득, 설명, 가이드 |
| **D2: 내러티브** | 소설, 시나리오, 에세이 | 창의적 글쓰기 |
| **D3: 시각 콘텐츠** | 인포그래픽, 다이어그램 | 시각적 표현 |
| **D4: 프레젠테이션** | 세미나 슬라이드, 대시보드 | 프레젠테이션 |
| **D5: 이미지 생성** | ComfyUI 프롬프트, 배치 | 이미지 파이프라인 |

| **Template Filler** | 템플릿 기반 생성 | 🟢 룰 |

---

## ✅ 검증 게이트 (Anti-Slop)

| Level | 체크 항목 | 예시 |
|-------|----------|------|
| **L1** | Schema 검증 | JSON/YAML 유효성 |
| **L2** | Error 체크 | 문법, 철자, 중복 |
| **L3** | Voice 일치 | 어조, 톤앤매너 |
| **L4** | Domain 적합성 | 전문 용어, 컨텍스트 |
| **L5** | Judge Model | 최종 품질 판정 |

### Anti-Slop 금지어 패턴

- "~가 아니라 ~이다" → "~이다"
- "~대신 ~을 사용" → "~을 사용"
- "delve", "Moreover", "In conclusion" → 제거
