# 왜 AI는 기억하지 못하는가

> 세션이 끝나면 모든 컨텍스트가 사라지는 문제와 Hermes의 해결책

---

## AI의 기억 상실 증후군

ChatGPT, Claude, Gemini — 오늘날의 AI는 대화가 끝나면 **아무것도 기억하지 못합니다**. 새 세션을 열면 모든 컨텍스트가 초기화되고, 에이전트는 사용자를 처음 만난 사람처럼 행동합니다.

### 세션 기반 AI의 문제

```
세션 1: "API 설계 도와줘"
→ AI: "좋아, RESTful API를 설계해보자"
→ 100턴 대화 끝에 완성

세션 2: "저번 API 설계 이어서 할게"
→ AI: "무슨 API요? 처음 듣는데요?"
```

이것이 현재 AI 시스템의 **최대 약점**입니다.

## 왜 AI는 기억하지 못하는가?

### 1. 통계적 한계

LLM은 본질적으로 **통계적 예측 모델**입니다. 주어진 컨텍스트(프롬프트)에서 가장 확률 높은 다음 토큰을 예측할 뿐, **영구적인 기억 구조**가 없습니다.

```
LLM의 작동 원리:
"사용자가 '안녕'이라고 말했어. 그다음에 뭐가 올까?"
→ "하세요"가 가장 확률 높음
→ "하세요" 출력
```

이것이 전부입니다. LLM은 학습 데이터에 저장된 패턴을 재현할 뿐, 개인화된 기억을 유지하는 능력이 없습니다.

### 2. 컨텍스트 윈도우의 물리적 한계

모든 LLM에는 **컨텍스트 윈도우**라는 한계가 있습니다:

| 모델 | 컨텍스트 윈도우 | 현실적 한계 |
|------|:-------------:|:----------:|
| GPT-4 | 128K 토큰 | ~300페이지 |
| Claude 3 | 200K 토큰 | ~500페이지 |
| Gemini | 1M 토큰 | ~2,500페이지 |

아무리 큰 컨텍스트 윈도우라도 **무한하지 않습니다**. 오래된 대화는 결국 밀려나고, 에이전트는 중요한 정보를 잊어버립니다.

### 3. 구조화된 저장소의 부재

채팅 인터페이스는 **비구조화된 대화 로그**만 저장합니다. "누가, 언제, 무엇을 결정했는지"와 같은 구조화된 정보를 저장하고 검색하는 메커니즘이 없습니다.

```
일반 채팅 저장소:
[12:01] 사용자: 데이터베이스는 PostgreSQL 사용하기로 했어
[12:02] AI: 좋은 선택입니다.
→ 구조화되지 않음 → 검색 불가 → 다음 세션에서 무효
```

## Hermes의 접근: 영속적 기억 시스템

Hermes는 이 문제를 **Knowledge 시스템**으로 해결합니다.

### 세션과 Knowledge의 분리

```
일반 AI: 세션 = 모든 것
Hermes:  세션 = 대화 흐름
           Knowledge = 영구 기억
```

세션은 단순한 대화 흐름일 뿐입니다. 진짜 기억은 **Knowledge**에 저장됩니다.

### Knowledge 3-Tier 구조

| 계층 | 수명 | 예시 |
|------|:----:|------|
| **T1 Facts** | 영구 | API 엔드포인트, 비밀번호 규칙 |
| **T2 Context** | 프로젝트 기간 | 아키텍처 결정, 설계 이유 |
| **T3 Reasoning** | 분석 기간 | 코드 리뷰 결과, 성능 분석 |

Knowledge는 세션과 독립적으로 저장되며, FTS5 검색으로 필요할 때 즉시 불러옵니다.

### Memory 시스템의 자동 저장

Hermes는 모든 대화를 **세션 DB**에 자동 저장합니다. 사용자가 별도로 저장할 필요 없이, 과거 대화를 검색할 수 있습니다.

```
session_search("PostgreSQL 결정")
→ "데이터베이스는 PostgreSQL 사용하기로 결정 (JOB-1234, 2025-01-15)"
```

## 비교: 일반 AI vs Hermes

| 항목 | 일반 AI | Hermes |
|------|---------|--------|
| **기억 지속성** | 세션 내에서만 | 영구적 |
| **검색 능력** | 스크롤만 가능 | FTS5 전문 검색 |
| **구조화** | 비구조화 채팅 로그 | T1/T2/T3 계층 구조 |
| **자동 저장** | 없음 | 전체 대화 자동 저장 |
| **크로스 세션** | 불가능 | session_search로 가능 |

---

## 결론: 기억은 선택이 아니라 필수

AI가 진정한 생산성 도구가 되려면 **기억해야 합니다**. 세션이 바뀌어도, 시간이 지나도 중요한 정보를 잊지 않는 시스템이 필요합니다.

Hermes의 Knowledge 시스템은 이 문제에 대한 실용적인 해결책입니다.
**기억하는 AI**는 단순히 똑똑한 AI보다 훨씬 가치 있습니다.
