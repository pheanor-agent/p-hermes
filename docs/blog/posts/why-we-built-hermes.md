# Hermes를 만든 이유

> 프로젝트 철학과 비전 — 우리는 왜 Hermes Agent를 만들었는가

---

## 문제의 시작

2024년, AI 에이전트가 폭발적으로 성장하던 시기. 수많은 AI 도구와 프레임워크가 등장했지만, 모두 **근본적인 문제**를 안고 있었습니다.

### 우리가 발견한 3가지 문제

**1. AI는 기억하지 못한다**
ChatGPT, Claude, Gemini — 모든 AI는 세션이 끝나면 아무것도 기억하지 못합니다. 사용자는 매번 같은 정보를 반복해서 알려줘야 합니다.

**2. AI는 체계적으로 실행하지 못한다**
자율성만 강조된 에이전트는 무분별하게 실행합니다. 조사 없이, 검증 없이, 감사 없이 — 결과는 예측 불가능합니다.

**3. AI는 배우지 못한다**
이전 작업의 성공과 실패가 다음 작업에 반영되지 않습니다. 매번 처음부터 시작하는 것과 다름없습니다.

---

## 왜 기존 솔루션이 부족했는가

| 솔루션 | 한계 |
|--------|------|
| **RAG (Retrieval Augmented Generation)** | 외부 데이터는 검색하지만 자체 기억 구조 없음 |
| **프롬프트 엔지니어링** | 수동 관리 필요, 확장 불가능 |
| **멀티 에이전트 프레임워크** | 복잡도만 증가, 품질 보증 메커니즘 부재 |
| **단일 에이전트 + 도구** | 기억도, 프로세스도, 학습도 없음 |

기존 솔루션은 문제의 **일부만 해결**했습니다.
Hermes는 **전체 문제**를 해결하고자 했습니다.

---

## Hermes의 철학

### 철학 1: 기억하라

AI가 기억하지 못하면 생산성 도구가 될 수 없습니다. Knowledge 시스템은 이 문제의 해결책입니다.

- 모든 대화가 세션 DB에 저장됨
- 중요한 정보는 Knowledge로 영구 보관
- FTS5 검색으로 필요할 때 즉시 검색

### 철학 2: 체계적으로 실행하라

자율성은 프로세스 위에서만 안전합니다. Workflow Gate는 이 원칙을 구현합니다.

- 9단계 품질 게이트
- 각 단계별 검증 기준
- 롤백 및 재시도 지원

### 철학 3: 배우고 성장하라

사용하면 사용할수록 더 똑똑해지는 시스템. Skill과 Knowledge가 이 순환을 만듭니다.

- 작업 산출물이 Knowledge로 축적
- 반복 패턴을 Skill로 저장
- 시간이 지날수록 시스템 진화

---

## 왜 "Hermes"인가

Hermes는 그리스 신화의 **신들의 전령**입니다. 메시지를 전달하고, 정보를 연결하고, 명령을 수행합니다.

AI 에이전트도 같은 역할을 해야 한다고 믿었습니다:
- **메시지 전달**: 다양한 플랫폼(Discord, Telegram, CLI) 간 메시지 연결
- **정보 연결**: 세션, Knowledge, Memory 간 정보 연결
- **명령 수행**: Job 기반 작업 실행 및 관리

---

## Hermes가 풀고자 하는 문제

### 1. "매일 반복되는 작업을 위임할 수 없을까?"
→ Cron Job + Skill로 해결

### 2. "과거에 했던 논의를 다시 찾을 수 없을까?"
→ session_search + Knowledge로 해결

### 3. "AI가 저지른 실수를 추적할 수 없을까?"
→ Workflow Gate + 산출물로 해결

### 4. "팀의 지식이 개인에게만 머무르지 않게 할 수 없을까?"
→ Knowledge 시스템 + Skill 공유로 해결

---

## 우리의 비전

> **Hermes는 단순한 AI 도구가 아닙니다.**
>
> Hermes는 **AI와 인간의 협업 방식을 근본적으로 바꾸는 플랫폼**입니다.

### 단기 비전
- 개인 개발자의 생산성 도구
- 반복 작업 자동화
- 지식 관리 체계화

### 장기 비전
- 팀 단위 협업 플랫폼
- 조직 지식 관리 시스템
- AI 에이전트 생태계의 표준 워크플로우

---

## 우리가 하지 않은 선택

### 우리는 API 서비스를 만들지 않았다
Hermes는 자체 호스팅(self-hosted) 도구입니다. API 서비스가 아니라, **사용자의 환경에서 실행되는 에이전트**입니다.

### 우리는 완전 자동화를 목표로 하지 않았다
100% 자동화는 현실적으로 불가능합니다. Hermes는 **사람이 개입해야 할 지점**을 명확히 하고, 나머지를 자동화합니다.

### 우리는 단순함을 희생하지 않았다
"AI = 복잡하다"는 공식을 깨고 싶었습니다. 개념은 간단하게, 내부는 강력하게.

- 신규 사용자의 학습 곡선을 낮춥니다

- 모범 사례와 패턴을 기록합니다
