핵심 개념 3가지

1

분리 (Separation)

관심사를 계층으로 분리. 설정은 설정, 실행은 실행, 통신은 통신 — 각각 독립적으로 진화

2

비동기 (Asynchronous)

이벤트 버스를 통한 Pub/Sub 통신. 호출자-수신자 결합 해방, 실패 격리

3

확장 (Extensibility)

Plugin 아키텍처. 기존 코드 수정 없이 새 인터페이스, 스킬, 크론 작업 추가

05 / 10

Core 계층
정책과 규칙의 근원

  • config.yaml: 전역 설정 — 모델, 프로바이더, 시스템 프롬프트 등 기본 정책 설정
  • AGENTS.md: 에이전트 역할 정의 — 행동 규범, 책임 범위, 제약 조건 명시 규칙
  • skills/: 스킬 정의 디렉토리 — 도구 사용법, 워크플로우, 프롬프트 템플릿 확장
config.yaml # 전역 설정 SSOT AGENTS.md # 에이전트 계약서 skills/ hermes-agent/ file-editor/ ... # 플러그인 가능한 스킬
06 / 10

Runtime 계층
실행 컨텍스트 관리

  • sessions/: 대화 세션 저장 — 프롬프트 히스토리, 컨텍스트 창, 상태 스냅샷 상태
  • cron/: 주기적 작업 스케줄 — background 프로세스, 예약된 에이전트 실행 정기
  • state 파일: JSONL 히스토리, 이벤트 로그 — 시스템의 모든 변경 추적 추적
sessions/ session-20260617-001.jsonl # JSONL 대화 로그 cron/ daily-report.yaml # 매일 09:00 실행 state/ events.jsonl # 이벤트 버스 로그
07 / 10

Interfaces 계층
통신 엔드포인트

  • Discord: Discord Bot 인터페이스 — 채널 기반 대화, 미디어 첨부, rich embed Discord
  • Telegram: Telegram Bot 인터페이스 — 인라인 키보드, 파일 전송, 그룹.chat Telegram
  • CLI / API: 터미널 인터페이스와 REST 엔드포인트 — 로컬 개발자 워크플로우 CLI
interfaces/ discord/ bot.py # Discord.py 기반 telegram/ bot.py # python-telegram-bot cli/ main.py # Typer CLI # 각 인터페이스는 Runtime 계층에만 의존

Infra / Release 계층 08 / 10

I Infra 계층

기반 환경 설정과 인프라 구성

  • .env 환경변수 — DB 연결, API 키, 포트
  • cron.d.schedule — 시스템 스케줄
  • infra/ — Docker Compose, Nginx conf
  • 네트워크 구성, SSL 인증서

R Release 계층

배포 파이프라인과 릴리즈 관리

  • GitHub Actions CI/CD 워크플로우
  • 테스트 → 빌드 → 배포 자동화
  • GitHub Pages 정적 사이트 호스팅
  • 버전 태깅과 CHANGELOG

C 배포 전략

Docker vs GitHub Pages

  • Docker: 전체 앱 스택 — 컨테이너화, 로컬 개발, 프로덕션
  • GitHub Pages: 문서 + 정적 — docs/, llms.txt, API 레퍼런스
  • hybrid: 앱은 Docker, 문서는 Pages — 각각 독립 배포
docker-compose.yml # 앱 컨테이너 정의 .github/workflows/ ci.yml # 테스트 deploy-docs.yml # Pages 배포 deploy-app.yml # Docker 배포

모듈화 아키텍처 가치 3가지

레이어드 구조 설계로 얻는 핵심 가치

1

독립적 진화

각 계층은 독립적으로 개발 · 테스트 · 배포됩니다. 변경 영향 범위가 명확히 격리됩니다.

2

확장 가능한 인터페이스

새 인터페이스 추가 시 코어 로직 수정 불필요. event.sh bus만 연결하면 됩니다.

3

실패 격리

한 인터페이스 또는 계층의 실패가 전체 시스템으로 전파되지 않습니다. 비동기 버퍼가 보호합니다.
