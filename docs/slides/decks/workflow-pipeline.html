Workflow Pipeline

9단계 상태머신 및 작업 흐름

9단계 워크플로우

  1. request
  2. investigation
  3. design
  4. review
  5. approval
  6. execution
  7. test
  8. execution_review
  9. done

상태 전이 규칙

  • 각 단계는 순차적으로 진행
  • review → approval: 사용자 승인 필요
  • execution 실패: error 상태로 전환
  • done: 결과 알림 전송

workflow-gate.sh


# 워크플로우 상태 관리
bash workflow-gate.sh JOB-1234 start
bash workflow-gate.sh JOB-1234 transition design
bash workflow-gate.sh JOB-1234 complete

  • 9단계 상태 머신
  • 승인 단계 강제
