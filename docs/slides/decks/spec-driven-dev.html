Spec 라이프사이클

proposed → draft → review → approved → deprecated
                ↑         ↓
                ←── 수정 ──┘
  • proposed: 초기 생성
  • draft: 작성 중
  • review: 리뷰 요청
  • approved: 승인 완료

Contract 정의

contract:
  precondition: [시스템 상태 A]
  postcondition: [시스템 상태 B]
  invariant: [항상 참인 조건]
  • Precondition: 실행 전 조건
  • Postcondition: 실행 후 조건
  • Invariant: 항상 유지되는 조건

Examples 정의

examples:
  - name: 사용 예시
    command: 실행 명령어
    expected: 기대 결과
  • 실제 사용 예시 정의
  • 명령어와 기대 결과 명시
  • Conformance Score 30% 가중

Traceability 패턴

# Spec: SPEC-A01
# Code: SPEC-A01 기능 구현
# Test: SPEC-A01 기능 검증
  • Spec → 코드 → 테스트 추적
  • 코드/테스트에 Spec ID 참조
  • 자동화된 Traceability 검증
