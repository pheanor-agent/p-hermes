챗봇의 3가지 한계

① 세션 종료 = 모든 것 소멸
어제 한 대화, 오늘 기억 못함. 매번 다시 설명해야 함.

HOW

어떻게 학습하는가?

GEPA 학습 루프

에이전트가 경험을 재사용 가능한 지식으로 변환하는 사이클

복잡한 작업을 해결한 후 자동으로 SKILL.md 생성 → 다음 세션에서 자동 로드

GEPA 단계별 상세

Gather실행한 명령어, 에러, 성공 워크플로우 기록
Evaluate5회 이상 도구 호출 / 에러 해결 / 반복 가능성 평가
ProduceYAML 프런트매터 + 마크다운 본문으로 스킬 문서화
Apply다음 세션에서 일치하는 스킬 자동 로드

→ 매번 같은 실수를 반복하지 않는 "성숙하는" 에이전트

실제 사용 예시


# 사용자가 Telegram에서 작업 요청
$ hermes chat

# 에이전트가 자율적으로 작업 수행
> "로그 파일에서 에러 패턴 분석해서 리포트 만들어줘"

# 1. 터미널로 로그 수집
# 2. 패턴 분석 (Python 스크립트)
# 3. 리포트 파일 생성
# 4. 결과를 Telegram으로 전송

# 완료 후 자동으로 스킬 생성 (GEPA)
# → 다음번 로그 분석 요청은 더 빠르게 수행

이런 경우에는 부적합

단순 Q&A만 필요한 경우
챗봇이면 충분. 에이전트는 오버스펙.

핵심 요약

  3. 20개 이상 제공자 지원, 벤더 중립
  4. 6개 핵심 시스템으로 신뢰성/품질 보장
