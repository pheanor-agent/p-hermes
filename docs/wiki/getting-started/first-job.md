# 첫 번째 작업 요청하기

Hermes Agent의 가장 강력한 특징은 단순한 대화가 아니라, **구조화된 작업(JOB)**을 수행한다는 점입니다. 이 가이드는 Hermes에게 첫 번째 공식 작업을 요청하고 완료하는 과정을 설명합니다.

## 🎯 JOB이란 무엇인가?

Hermes에게 작업을 요청하면, 에이전트는 이를 단순한 채팅이 아닌 **추적 가능한 프로젝트**로 인식합니다.

- **추적성**: 모든 작업은 고유 ID (예: JOB-1001)를 가지며, 전용 작업 폴더에 모든 기록이 남습니다.
- **신뢰성**: 9단계 워크플로우 (조사 → 설계 → 검증 → 실행 → 완료)를 거쳐 결과물을 생산합니다.
- **재현성**: 작업 과정에서 생성된 설계서와 결과물이 보존되어 나중에 언제든 복기할 수 있습니다.

> **참고**: `[JOB-XXXX]` 형식은 선택사항입니다. 자연어로 요청하면 에이전트가 자동으로 JOB 번호를 부여합니다.

---

## 🚀 단계별 요청 방법

### 1. 작업 요청 보내기

채팅창에 명확한 목표를 입력하세요. `[JOB-번호]` 형식은 선택사항이며, 에이전트가 자동으로 JOB 번호를 부여합니다.

**예시 요청 (자연어)**:
> 현재 프로젝트의 모든 .md 파일을 분석해서 누락된 링크를 찾고 리포트를 작성해줘

**예시 요청 (JOB 번호 명시)**:
> JOB-1001로 현재 프로젝트의 .md 파일 분석해서 누락된 링크 찾아줘

**두 방식 모두 동일하게 처리됩니다.** 에이전트는 요청을 분석하여 JOB 번호를 부여하고, 9단계 워크플로우로 진행합니다.

### 2. 진행 상황 확인 (Workflow)

요청을 보내면 Hermes는 다음과 같은 단계로 작업을 진행하며, 각 단계가 끝날 때마다 상태를 보고합니다.

1. **Investigation**: 요청 분석 및 현황 파악
2. **Design**: 해결 방안 및 상세 설계서 작성
3. **Review**: 설계서의 무결성 및 정책 준수 검토
4. **Approval**: 사용자에게 설계 승인 요청 (**여기서 사용자의 "진행해"라는 승인이 필요합니다**)
5. **Execution**: 실제 코드 작성 및 파일 수정
6. **Test**: 결과물 검증 및 테스트
7. **Done**: 최종 결과 보고 및 지식 동기화

### 3. 결과 확인 및 승인

작업이 완료되면 Hermes가 최종 결과물 (리포트, 수정된 파일 등)을 제공합니다. 결과가 만족스럽다면 작업 종료를 승인하고, 수정이 필요하다면 추가 요청을 보냅니다.

---

## 💡 더 나은 결과를 얻기 위한 팁

| 좋은 요청 (Good) | 부족한 요청 (Bad) | 이유 |
|---|---|---|
| 현재 프로젝트의 A 모듈 API 명세서를 작성하고, Pytest 테스트 코드 3개 작성해줘 | 이거 좀 수정해줘 | 목표가 구체적이고 검증 가능한 산출물이 명시됨 |
| README의 내용을 최신 아키텍처 (5-Tier)에 맞게 수정하고, 링크 무결성을 검증해줘 | 문서 좀 고쳐줘 | 변경 기준 (5-Tier)과 검증 방법 (링크 무결성)이 명확함 |

---

## ⚠️ 주의사항

- **승인 대기**: Hermes는 `Design` 단계 이후 반드시 사용자의 승인을 기다립니다. 응답이 없으면 작업은 `Approval` 단계에서 멈춥니다.
- **JOB 번호**: 기존에 진행 중인 JOB 번호를 사용하면 해당 작업의 연장선으로 처리됩니다.
