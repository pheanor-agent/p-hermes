## Content System이란?

Content System은 Hermes의 **콘텐츠 생성 엔진**입니다. 
단순히 텍스트를 생성하는 것이 아니라, **도메인별 가이드라인에 따라 구조화된 콘텐츠**를 만듭니다.

### Content System의 핵심 요소

1. **Pre-Direction Engine**: 작업 전 도메인 가이드를 주입하여 방향성을 설정
2. **도메인별 가이드**: 코딩, 문서, 분석 등 영역별 전용 지침
3. **품질 검증**: 생성된 콘텐츠의 일관성과 완성도 검증

## 콘텐츠 생성하기

### 기본 콘텐츠 생성

가장 간단한 형태의 콘텐츠 생성:

```
짧은 릴리스 노트를 작성해줘
```

Hermes가 Content System을 활성화하여 릴리스 노트 형식에 맞게 생성합니다.

### 도메인 지정

명시적으로 도메인을 지정할 수 있습니다:

```
/content 문서 "새로운 API 엔드포인트 문서를 작성해줘"
```

지원하는 도메인:
| 도메인 | 설명 | 예시 |
|--------|------|------|
| `문서` | 기술 문서 | API 문서, 가이드 |
| `코드` | 소스 코드 | 기능 구현, 리팩토링 |
| `분석` | 데이터 분석 | 로그 분석, 성능 평가 |
| `블로그` | 블로그 포스트 | 기술 블로그, 회고 |

## Pre-Direction Engine 활용

Pre-Direction Engine은 콘텐츠 생성 **전에** 방향성을 설정합니다.

### 간단한 방향 설정

```
이 문서는 초보 개발자 대상으로 쉽게 설명해줘
```

### 상세한 방향 설정

```
이 문서는 다음과 같은 기준으로 작성해줘:
- 대상: 신규 팀원
- 목적: 빠른 온보딩
- 톤: 친절하지만 전문적
- 분량: 5분 이내에 읽을 수 있는 분량
```

## 콘텐츠 품질 관리

Content System은 생성된 콘텐츠의 품질을 자동으로 검증합니다.

### 검증 항목
- **구조 완성도**: 필수 섹션이 모두 포함되었는가
- **일관성**: 도메인 가이드와 일치하는가
- **가독성**: 대상 독자에게 적합한 수준인가
- **정확성**: 사실과 일치하는가 (Knowledge 검증)

---

## 실전 예제

### 예제 1: 기술 문서 생성

```
요청: "데이터베이스 마이그레이션 가이드를 작성해줘"
→ Content System 활성화
→ Pre-Direction Engine이 문서 구조 설정
→ Knowledge 검색으로 관련 정보 수집
→ 구조화된 문서 생성
→ 품질 검증
→ 완료
```

### 예제 2: 릴리스 노트 생성

```
요청: "이번 스프린트 릴리스 노트를 작성해줘"
→ Content System이 릴리스 노트 템플릿 적용
→ 변경 사항 목록 자동 수집
→ 버전, 날짜, 변경 내역 구조화
→ 마크다운 형식으로 출력
```

## Content System의 장점

| 장점 | 설명 |
|------|------|
| **일관성** | 모든 콘텐츠가 동일한 품질 기준으로 생성 |
| **효율성** | 반복적인 형식 작업을 자동화 |
| **확장성** | 새로운 도메인 가이드를 추가하여 확장 |
| **학습** | 사용자의 선호도를 학습하여 점진적 개선 |
