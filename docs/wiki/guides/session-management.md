# 🔄 세션 관리

> Hermes 세션의 시작, 전환, 검색 방법을 다룹니다.
> 세션은 대화의 단위이며, Knowledge는 세션을 초월하여 유지됩니다.

---

## 세션이란?

세션은 Hermes와의 **연속된 대화 흐름**입니다. 
각 세션은 고유한 ID를 가지며, 모든 대화 내용이 자동으로 저장됩니다.

### 세션의 특징
- **자동 저장**: 모든 대화가 DB에 기록됩니다
- **컨텍스트 유지**: 같은 세션 내에서는 맥락이 유지됩니다
- **검색 가능**: 과거 세션을 검색하여 참조할 수 있습니다
- **플랫폼 독립**: Discord, Telegram, CLI 등 모든 플랫폼에서 동일하게 동작합니다

---

## 세션 시작

### 자동 시작

첫 메시지를 보내면 자동으로 세션이 생성됩니다:

```
사용자: "안녕하세요"
→ 새 세션 생성 (session_abc123)
```

### 명시적 시작

```
/new
→ 새로운 세션이 시작되었습니다
```

### JOB 기반 세션

```
JOB으로 만들어줘: 데이터베이스 마이그레이션
→ JOB-1234 생성
→ JOB-1234 전용 세션 시작
```

---

## 세션 전환

### 현재 세션 확인

```
/sessions
→ 현재 세션: session_abc123
  최근 5개 세션:
  1. session_abc123 (현재) - "데이터베이스 마이그레이션"
  2. session_def456 - "API 문서 작성"
  3. session_ghi789 - "코드 리뷰"
```

### 세션 열기

```
/sessions open session_def456
→ session_def456 세션으로 전환

/sessions open JOB-1234
→ JOB-1234 세션으로 전환
```

## 세션 검색

### 기본 검색

```
/sessions search "API 배포"
→ 검색 결과:
  1. session_jkl012 - "API 배포 스크립트 작성" (3일 전)
  2. session_mno345 - "배포 파이프라인 설정" (2주 전)
```

### 고급 검색

명령어를 사용한 고급 검색:

```
session_search("API 배포 설정")
→ FTS5 기반 전문 검색 결과 반환
```

## 세션과 플랫폼의 관계

```
사용자 (Discord) ───→ Hermes ───→ 세션 DB
사용자 (Telegram) ──→ Hermes ───→ 세션 DB
사용자 (CLI) ───────→ Hermes ───→ 세션 DB
```

모든 플랫폼에서의 대화가 **하나의 세션 DB**에 저장됩니다.
즉, Discord에서 시작한 작업을 CLI에서 이어받을 수 있습니다.

---

## 세션 관리 팁

### 팁 1: 세션은 컨텍스트 단위
너무 오래된 세션은 컨텍스트가 오염될 수 있습니다. 주제가 바뀌면 `/new`로 새 세션을 시작하세요.

### 팁 2: JOB이 더 강력하다
단순한 대화는 세션으로 충분하지만, 추적이 필요한 작업은 반드시 JOB으로 만드세요. JOB은 세션보다 더 구조화된 관리를 제공합니다.

### 팁 3: Knowledge를 세션 공용 저장소로
세션 간에 정보를 공유해야 한다면 Knowledge에 저장하세요. 세션이 바뀌어도 Knowledge는 유지됩니다.

> **💡 Tip**: 세션 관리는 Hermes 사용의 기본입니다. 주제별로 세션을 분리하고, 중요한 정보는 Knowledge로 이관하세요.
