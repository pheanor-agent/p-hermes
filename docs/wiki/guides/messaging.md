# 메시징 플랫폼 연동

Hermes는 사용자의 주요 대화 장소에 직접 통합됩니다. Discord와 Telegram을 연동하여 언제 어디서든 에이전트와 소통할 수 있습니다.

## 🤖 Discord 연동

Hermes는 Discord 봇으로 운영됩니다.

- **서버 홈 채널**: 지정된 디스코드 서버에서 `/` 명령어 또는 멘션(`@HermesBot`)을 통해 호출합니다.
- **쓰레드(Threads)**: `[JOB-XXXX]` 작업이 생성되면 에이전트는 진행 상황과 결과를 별도의 쓰레드에 기록하여 메인 채널을 깔끔하게 유지합니다.
- **승인 요청**: 설계서 승인이 필요할 때, Hermes는 현재 대화 중인 쓰레드로 직접 승인 요청 메시지를 보냅니다.

---

## 📱 Telegram 연동

Telegram 그룹에서도 Hermes를 호출할 수 있습니다.

- **그룹 연동**: Hermes가 가입된 텔레그램 그룹에서 직접 메시지를 보낼 수 있습니다.
- **크론 결과 전송**: 자동화 작업(Cron)의 결과물을 특정 텔레그램 그룹으로 바로 전송하도록 설정할 수 있습니다.

---

## 🔗 플랫폼 간 통신 규칙

Hermes는 플랫폼에 상관없이 동일한 워크플로우를 따릅니다.

1. **수신**: Discord나 Telegram에서 요청을 수신합니다.
2. **처리**: 로컬 서버에서 작업을 실행합니다.
3. **송신**: 결과를 원래 요청이 들어온 플랫폼(채널)으로 회신합니다.

## 💡 팁
- **이미지 첨부**: Hermes는 Discord나 Telegram에 이미지나 PDF 파일을 직접 첨부하여 보낼 수 있습니다.
- **스레드 활용**: 디스코드에서는 쓰레드를 활용하면 여러 작업을 동시에 추적하기 훨씬 수월합니다.
