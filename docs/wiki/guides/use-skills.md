# 스킬 시스템 활용하기

Hermes의 핵심 기능인 **스킬(Skill)**은 에이전트가 특정한 작업을 수행할 때 참조하는 '전문가 지식'입니다. 스킬을 활용하면 에이전트의 작업 정확도와 속도를 극대화할 수 있습니다.

## 🧠 스킬이란?

에이전트는 방대한 데이터를 모두 기억하지 못합니다. 따라서 특정 작업이 발생했을 때, 해당 작업에 최적화된 **SKILL.md** 파일을 동적으로 로드하여 지침을 따릅니다.

- **활성화 조건**: 요청에 관련 키워드가 있거나 에이전트가 판단할 때 자동으로 로딩됩니다.
- **구조**: 하나의 스킬은 `SKILL.md`와 참조 파일(`references/`, `templates/`, `scripts/`)로 구성됩니다.
- **규모**: Hermes는 현재 **78개 이상의 내장 스킬**과 수백 개의 오픈소스 스킬을 보유하고 있습니다.

---

## 🛠️ 스킬 관리 방법

### 1. 스킬 목록 확인
사용자는 Hermes에게 다음과 같이 요청하여 현재 설치된 스킬 목록을 확인할 수 있습니다.
> "현재 설치된 스킬 목록을 보여줘."

### 2. 내만의 스킬 작성하기 (커스텀 스킬)
반복적인 작업을 시스템화하고 싶다면 직접 스킬을 작성할 수 있습니다.

**예시: `[TASK] Git 커밋 메시지 작성 스킬을 만들어줘`** 요청 시 에이전트는 다음을 수행합니다:
1. 새로운 스킬 폴더를 생성합니다.
2. `SKILL.md`를 작성합니다 (프론트마터, 가이드라인, 검증 스크립트 등).
3. 스킬을 시스템에 등록합니다.

**기본 `SKILL.md` 템플릿:**
```markdown
# 스킬 이름

> 설명: 이 스킬의 목적

## 1. 언제 사용하는가? (Trigger)
- (사용 조건)

## 2. 수행 단계 (Steps)
1. ...
2. ...

## 3. 검증 방법 (Verification)
- ...
```

## 🧩 주요 스킬 카테고리

| 카테고리 | 대표 스킬 | 용도 |
|---|---|---|
| **Software Dev** | `test-driven-development`, `spec-driven-dev` | TDD, Spec-Driven 워크플로우 |
| **Creative** | `comfyui`, `p5js` | 이미지/비주얼 생성 |
| **Research** | `arxiv`, `polymarket` | 논문 검색, 시장 데이터 분석 |
| **Data Science** | `jupyter-live-kernel` | Jupyter 인터랙티브 분석 |
| **Productivity** | `google-workspace`, `notion` | 문서 및 외부 서비스 연동 |

---

## 🌍 외부 스킬 설치

커뮤니티에서 공유된 스킬을 설치하여 에이전트의 기능을 확장할 수 있습니다.
> "이 GitHub 저장소의 스킬을 설치해줘: `https://github.com/...`"

에이전트는 저장소를 클론하여 `~/.hermes/skills/`에 자동으로 등록합니다.
