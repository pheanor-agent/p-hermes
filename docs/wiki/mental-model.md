## Hermes ≠ ChatGPT, Hermes ≠ Claude

Hermes Agent는 일반적인 AI 채팅봇과 **근본적으로 다릅니다**.

| 항목 | ChatGPT / Claude | Hermes Agent |
|------|-----------------|--------------|
| **사용 방식** | 대화 기반, 질문-응답 | **Job 기반**, 작업-처리-완료 |
| **상태** | 각 세션이 독립적 | 세션 + 영속적 Knowledge + Workflow |
| **기억** | 세션 컨텍스트만 유지 (창 제한) | 전역 Knowledge 검색, Memory 시스템 |
| **프로세스** | 사용자가 직접 관리 | Workflow Gate가 자동 관리 |
| **재사용성** | 프롬프트 복사/붙여넣기 | Skill로 저장하여 재사용 |

Hermes는 **"질문하면 답하는 도구"가 아니라**, **"작업을 위임하고 추적하는 시스템"**입니다.

## Hermes = Memory + Workflow + Knowledge + Jobs

Hermes는 네 가지 핵심 요소로 구성됩니다:

```
Hermes = Memory (기억)
       + Workflow (프로세스)
       + Knowledge (지식)
       + Jobs (작업)
```

### 1. Memory — 영속적 컨텍스트
Hermes는 당신과의 대화를 **SQLite 데이터베이스**에 저장합니다. 세션이 끝나도 이전 대화 내용을 검색할 수 있습니다. 단순한 채팅 히스토리가 아니라, **에이전트가 참조할 수 있는 구조화된 기억**입니다.

### 2. Workflow — 체계적인 프로세스
모든 작업은 **9단계 Workflow Gate**를 통과합니다. 무분별한 실행을 막고, 각 단계에서 산출물을 검증합니다. 이는 품질 보증의 핵심 메커니즘입니다.

### 3. Knowledge — 장기 지식
Knowledge는 Hermes의 **장기 기억 저장소**입니다. 문서, 코드 조각, 설정 정보, 학습한 패턴을 저장하고 FTS5 검색으로 필요할 때 즉시 불러옵니다.

### 4. Jobs — 추적 가능한 작업
모든 요청은 **JOB**으로 등록됩니다. JOB에는 상태, 우선순위, 담당자가 할당되며, 진행 상황을 추적할 수 있습니다.

## "ChatGPT처럼 쓰면 안 된다"

Hermes를 처음 사용할 때 가장 흔한 실수는 **ChatGPT처럼 대화하듯 사용하는 것**입니다.

### ❌ ChatGPT 스타일 (잘못된 예)
```
"파이썬 코드를 리뷰해줘"
"좋아, 그럼 이제 배포해볼까?"
"아니, 그 전에 테스트 먼저 추가해"
```
→ 세션 컨텍스트에만 의존, 추적 불가, 재현 불가

### ✅ Hermes 스타일 (올바른 예)
```
/new "코드 리뷰 + 테스트 추가 + 배포"
→ JOB-1234 생성
→ 9단계 Workflow Gate가 자동 처리
→ 각 단계 산출물 저장
→ 완료 후 결과 전달
```
→ Job 기반, 추적 가능, 재현 가능, Knowledge에 산출물 자동 저장

## 세션 기반이 아닌 Job 기반 사고방식

Hermes는 **세션 중심이 아니라 Job 중심**으로 설계되었습니다.

### 세션과 Job의 차이
| 개념 | 세션 (Session) | Job |
|------|---------------|-----|
| **정의** | 실시간 대화 흐름 | 공식 작업 단위 |
| **수명** | 사용자가 종료할 때까지 | 완료 또는 취소될 때까지 |
| **추적** | 비공식 (히스토리만) | 상태 기계로 관리 |
| **산출물** | 채팅 텍스트 | 구조화된 문서 (조사/설계/결과) |
| **재현** | 불가능 (동일 입력 보장 불가) | 가능 (JOB ID로 전체 이력 조회) |

### Job 기반 사고방식 예시

**일상적인 요청도 Job으로:**
```
→ "JOB으로 만들어줘: 매주 월요일 오전 10시에 서버 상태 보고서 생성"
→ Cron Job 자동 등록
→ 매주 월요일 자동 실행 → Telegram으로 보고서 전송
```

**복잡한 작업도 Job으로:**
```
→ "JOB으로 만들어줘: 새 API 엔드포인트 설계부터 배포까지"
→ 조사 → 설계 → 검토 → 승인 → 실행 → 테스트 → 배포
→ 각 단계마다 검증 게이트 통과
```

## 신규 사용자 체크리스트

Hermes를 처음 시작할 때 다음 순서를 권장합니다:

1. **Mental Model 이해** (이 문서) — 10분
2. **설치 및 환경 설정** — 10분
3. **첫 Knowledge 저장** — 5분
4. **첫 Job 요청** — 10분
5. **첫 Content 생성** — 5분
6. **Skill 저장해보기** — 10분

> **💡 Tip**: Hermes는 배우는 데 시간이 필요한 도구입니다. ChatGPT처럼 즉시 답을 얻는 도구가 아니라, **함께 성장하는 시스템**이라고 생각하세요.
