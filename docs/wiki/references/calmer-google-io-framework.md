# CALMER + Google I/O 하이브리드 프레임워크

> **목적**: 기술 슬라이드 콘텐츠 구성을 체계화하여 청중의 학습 곡선과 이해도를 최적화
> **적용**: JOB-1691 슬라이드 재구성 작업에서 채택

## 한 줄 요약

CALMER (인지 부하 관리) + Google I/O (문제→해결→트레이드오프)를 결합한 하이브리드 프레임워크로, 기술 슬라이드의 WHY-WHAT-HOW 구조를 20-50-30 비율로 체계화하여 청중 학습 효과를 극대화합니다.

## 기본 개념

기술 프레젠테이션에서 청중의 인지 부하를 관리하고, 문제 상황부터 트레이드오프까지 체계적으로 전달하는 콘텐츠 구성 방법론입니다. 두 프레임워크를 결합하여 상호 장점을 보완합니다.

---

## 문제 상황

기술 문서나 슬라이드를 만들 때 다음 문제들이 반복적으로 발생합니다.

- **정보 과부하**: 한 슬라이드에 지나치게 많은 내용이 담겨 있어 청중이 핵심을 파악하지 못함
- **단방향 설명**: WHAT만 나열하고 WHY (배경)와 HOW (실제 적용)가 누락됨
- **트레이드오프 누락**: 모든 도구에 장단점이 있음에도 장점만 소개하여 선택 기준이 부재
- **시각화 부족**: 텍스트 중심 설명이 많아 복잡한 구조를 이해하기 어려움

---

## 기술 설계

CALMER 프레임워크를 기반으로 합니다.

| 단계 | 의미 | 목적 |
|------|------|------|
| **C**ontext | 문맥 설정 | 개념의 배경과 필요성 제시 |
| **A**nchor | 닻 | 실제 사례나 비유로 개념 고정 |
| **L**ayering | 층층이 쌓기 | 쉬운 예시 → 다이어그램 → 코드 순서로 난이도 상승 |
| **M**emory Hook | 기억 고리 | 핵심 요약을 반복하여 장기 기억으로 전환 |
| **E**xercise | 연습 | 청중이 직접 적용해보는 경험 |
| **R**eview | 검토 | 복습과 연결 |

## 구조/흐름도

하이브리드 조합은 CALMER의 인지 부하 관리 + Google I/O의 문제→해결→트레이드오프 구조를 조합합니다.

```
WHY (20%)        문제 상황 + 문맥 설정 (CALMER: Context + Anchor)
    ↓
WHAT (50%)       개념 소개 + 시각화 + 코드 (CALMER: Layering + Google I/O: Solution)
    ↓
HOW (30%)        실제 적용 + 트레이드오프 + 예시 (CALMER: Exercise + Review + Google I/O: Demo/Trade-offs)
```

### 구조 비율

| 섹션 | 비중 | 포함 내용 |
|------|------|----------|
| **WHY** | 20% | 문제 상황, 배경, 왜 필요한지 |
| **WHAT** | 50% | 핵심 개념, 아키텍처 다이어그램, 코드 예시 |
| **HOW** | 30% | 적용 방법, 트레이드오프, 실제 사례 |

## 활용 예시

JOB-1691에서 8개 슬라이드 덱 전량에 적용했습니다.

### Before (기존 방식)

```
슬라이드 1: 아키텍처 소개
슬라이드 2: 레이어 설명
슬라이드 3: 데이터 흐름
슬라이드 4: 장점
슬라이드 5: 결론
```
- 문제 상황: 0%
- 예시: 0%
- 다이어그램: 0%
- 트레이드오프: 0%

### After (CALMER + Google I/O)

```
Slide 1-3:   WHY - 모놀리식 한계, 실제 문제 상황 (20%)
Slide 4-10:  WHAT - 레이어별 역할, 아키텍처 다이어그램, 이벤트 버스 (50%)
Slide 11-16: HOW - 디렉토리 구조, 데이터 흐름 예시, 트레이드오프 (30%)
```
- 문제 상황: ✅ 명시
- 예시: ✅ 실제 코드와 디렉토리 구조
- 다이어그램: ✅ Mermaid + ASCII
- 트레이드오프: ✅ 적용 조건 명시
