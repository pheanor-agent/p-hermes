# 자주 묻는 질문 (FAQ)

Hermes 사용 중 자주 발생하는 질문과 해결 방법을 모았습니다.

## 🛠️ 일반적 질문

### Q1. Hermes에게 작업을 지시할 때 꼭 `[JOB-XXXX]`를 써야 하나요?
- **필수는 아니지만 강력히 권장합니다.**
- `[JOB-XXXX]`를 사용하면 Hermes가 작업을 단순 채팅이 아닌 공식 프로젝트로 인식하여 설계서 작성, 상태 추적, 결과 보관 등 **9단계 워크플로우**를 따릅니다.
- 간단한 작업(예: "이 문장을 번역해줘")은 `[TASK]` 또는 일반 채팅으로 충분합니다.

### Q2. Hermes의 응답이 느릴 때는 어떻게 해야 하나요?
- 현재 수행 중인 작업이 복잡하거나(예: 코드베이스 전수 조사), 모델이 무거울 수 있습니다.
- `[FEEDBACK] 진행 상황좀 알려줘`라고 입력하면 현재 단계를 알려줍니다.
- `Investigation`이나 `Design` 단계라면 잠시 기다려주는 것이 좋습니다.

### Q3. 실행 중 오류가 났을 때 어떻게 하나요?
- Hermes는 자체적으로 `Test` 단계를 통과해야 `Done`으로 마무리합니다. 만약 실패하면 에이전트가 스스로 롤백하거나 재시도합니다.
- 사용자가 직접 개입해야 할 경우: `[FEEDBACK] X 오류가 발생해서 멈췄어. 해결해줘`라고 요청하세요.

---

## 🧠 지식 및 설정

### Q4. Hermes가 "정보가 없습니다"라고 하는데, 내가 전에 말한 건데 왜 모르지?
- Hermes는 **세션(Conversation)** 단위로 기억합니다. 아주 오래된 세션은 데이터베이스로 архив됩니다.
- "과거 세션에서 'X'에 대해 논의했던 내용을 찾아줘"라고 요청하면 에이전트가 지식 베이스(`session_search`)에서 데이터를 복원해 줍니다.

### Q5. 모델을 직접 바꿀 수 있나요?
- **네, 가능합니다.** `config.yaml`의 `routing` 설정을 수정하거나, 에이전트에게 "이 작업부터 Y 모델로 바꿔줘"라고 지시하면 해당 세션부터 새로운 모델을 사용합니다.
- (예: `Design` 단계에는 추론이 강한 모델을, `Execution`에는 코드 처리가 빠른 모델을 사용)

## 📂 파일 및 데이터

### Q6. 작업 결과물이 어디에 저장되나요?
- 공식적인 `[JOB-XXXX]` 작업은 `~/.hermes/workspace/jobs/JOB-XXXX/` 폴더에 저장됩니다.
- 결과 파일(`result.md`), 설계서(`design.md`), 테스트 리포트 등이 이 폴더 안에 생성됩니다.

### Q7. 실수로 파일을 삭제했는데 복구 가능한가요?
- Hermes는 삭제 전 자동으로 `pre-delete-backup.sh`를 실행합니다.
- "X 파일을 실수로 지웠어. 백업에서 복구해줘"라고 요청하면 `~/.hermes/backups/`에서 파일을 되살려줍니다.

---

**이 문서에 없는 문제가 있나요?**
디스코드나 Telegram 채널에서 `[FEEDBACK] 문제 발생` 태그와 함께 증상을 알려주세요. Hermes가 즉각적으로 대응합니다.
