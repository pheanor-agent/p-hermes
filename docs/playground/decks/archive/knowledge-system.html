Knowledge System

구조화된 AI 지식 관리

3계층 품질 관리 · FTS5 전역 검색 · cron 자동 동기화
WHY WHAT HOW
WHY · 1/3

🧠 기억의 한계

AI 세션이 종료되면 컨텍스트가 증발한다 — 대화 내용이 영구 저장되지 않는다.

세션 경계에서 모든 맥락이 소멸 → 같은 질문을 반복하게 되고, 이전에 발견한 인사이트가 사라진다. 결국 같은 내용을 다시 조사하는 비용이 누적된다.

WHY · 2/3

🔍 검색 불가능

과거에 작성한 문서나 기록을 찾으려면 파일을 하나씩 열어봐야 한다.

grep 기반 탐색의 한계 — 파일 시스템을 샅샅이 뒤져도 원하는 내용이 어디 있는지 알 수 없다. FTS5와 같은 전문 검색 엔진이 없으면 지식은 묻힌다.

WHY · 3/3

♻️ 중복 지식 & 근본 원인

같은 내용을 반복 작성하고, 버전 관리 없이 문서가 흩어진다.

동일 주제 문서가 여러 곳에 분산 → 모순 발생, 어느 것이 최신인지 판단 불가

🔑 근본 원인: 지식의 체계적인 구조화 부재 — 계층, 태그, 검색 인덱스가 없어 지식이 자산이 아닌 휘발성 데이터로 남는다.

WHAT · 1/3

📊 3계층 구조 (T1 / T2 / T3)

문서 품질에 따라 세 단계로 분류하여 신뢰도와 신선도의 균형을 유지한다.

🔴

T1: 핵심 참조

불변 — 검증된 최고 품질 문서. 승격 심사 후 편집 잠금.

🟡

T2: 세부 가이드

반안정 — 활발히 개선 중. 점수 기반으로 T1 승격 후보.

🟢

T3: 초안 / 실험

휘발 — 빠른 기록용. 미검증, 점수 미달 시 자동 정리.

WHAT · 2/3

🔎 FTS5 전역 검색

SQLite FTS5로 모든 문서를 인덱싱하여 밀리초 단위 전문 검색을 제공한다.

⚡

초고속 검색

FTS5 역색인 — 수천 건의 문서에서도 <100ms 응답

🎯

정확도

형태소 분석 + 순위화 — 찾고자 하는 내용이 정확히 검색됨

WHAT · 3/3

🔄 자동 동기화

cron 기반 5분 간격 자동 동기화로 문서 변경 사항을 실시간 반영한다.

⏱️

5분 주기

변경 감지 → FTS5 재인덱싱 → 점수 재산정 → 배포

🤝

Git 연동

문서 저장소 변경 시 자동 반영, 충돌 시 백업 보관

HOW · 1/3

📝 Wiki 작성

마크다운 기반 문서 작성, T1 ~ T3 계층 분류를 명시하여 품질 기준을 확보한다.

1
문서 생성

Markdown + YAML frontmatter (제목, 태그, 계층, 작성일)

2
계층 분류

T1/불변, T2/반안정, T3/초안 중 문서 성격에 맞게 지정

3
메타데이터

태그, 카테고리, 연관 문서 링크를 frontmatter에 기입

HOW · 2/3

🏆 Scoring (build-scores.sh)

자동 점수 산정으로 품질 등급을 결정하고 T1 승격 기준을 마련한다.

📏

완성도

문서 길이, 섹션 수, 이미지/코드 포함 여부

🔗

연결성

내부 링크 수, 참조 문서 수, 백링크 존재 여부

📅

신선도

최종 수정일, 업데이트 빈도, 리뷰 이력

HOW · 3/3

✅ 검증 (wiki-validate.sh)

링크 무결성, 메타데이터 정합성, 계층 규칙 준수를 자동 검사한다.

🔗

링크 검증

내부 참조 및 외부 URL 데드링크 탐지

📋

메타 검증

YAML frontmatter 필수 항목 누락 여부 확인

📊

계층 검증

T1/T2/T3 규칙 준수 — T1은 승급 심사 통과 여부 확인

Pros

✅ 장점

전역 검색: 모든 문서가 FTS5 인덱싱되어 초고속 전문 검색 가능 — 찾는 데 <1초

자동 점수: build-scores.sh로 품질 등급 자동 산정 — 문서 신뢰도 시각화

중복 방지: T2→T1 승격 과정에서 문서 병합/정리 — 저장소가 깔끔하게 유지

Cons

⚠️ 한계

초기 구축 비용: 3계층 분류 체계 + FTS5 인덱스 설정 + cron 배포 파이프라인 구성 필요

수동 유지보수: T2→T1 승격은 자동화하기 어렵고 결국 사람이 판단해야 함

T1 기준 모호: 점수만으로 불변 대상 선정이 어려움 — 도메인 특화 기준 별도 정의 필요

요약

📝 3줄 요약

1

구조화된 지식 = 검색 가능한 자산 — 컨텍스트 증발을 방지하고 반복 조사 비용을 제거한다

2

3계층 (T1/T2/T3) 품질 관리로 신뢰도와 신선도의 균형을 유지한다

3

cron 5분 자동 동기화 + FTS5 전역 검색으로 최신 상태를 유지하고 즉시 찾는다
