💡 WHAT — Knowledge System 구성

📊
3계층 구조 (T1/T2/T3)
T1: 핵심 참조 (불변)
T2: 세부 가이드 (반안정)
T3: 초안/실험 (휘발)

모든 문서를 SQLite FTS5 인덱싱
초고속 전문 검색 지원
🔄
자동 동기화
cron 5분 간격 자동 동기화
변경 사항 실시간 반영
section_flow[3] · HOW

🔧 HOW — 운영 워크플로우

1
Wiki 작성

마크다운 기반 문서 작성, T1~T3 계층 분류 지정

2
Scoring (build-scores.sh)

자동 점수 산정 → 품질 등급 결정, T1 승격 기준

3
검증 (wiki-validate.sh)

링크 무결성, 메타데이터, 계층 규칙 준수 확인

section_flow[4] · TRADE-OFFS

⚖️ Trade-offs

✅ 장점

  • 전역 검색: 모든 문서 FTS5 인덱싱으로 즉시 검색
  • 자동 점수: build-scores.sh로 품질 등급 자동 산정
  • 중복 방지: T2→T1 승격 과정에서 문서 병합/정리

⚠️ 한계

  • 초기 구축 비용: 3계층 분류 + FTS5 설정 필요
  • 수동 유지보수: T2→T1 승격은 사람이 판단해야 함
  • T1 기준 모호: 점수만으로 불변 대상 선정이 어려움
section_flow[5] · SUMMARY
