## 도구 시스템 개요

Hermes는 다양한 작업을 수행하기 위해 **25개 이상의 내장 도구**를 제공합니다. 도구는 카테고리별로 그룹화되어 있으며, 필요에 따라 활성화/비활성화할 수 있습니다.

### 도구 카테고리

| 카테고리 | 도구 수 | 주요 용도 |
|----------|:-------:|-----------|
| **파일** | 3개 | 파일 읽기/쓰기/검색 |
| **터미널** | 2개 | 명령어 실행/프로세스 관리 |
| **브라우저** | 7개 | 웹 탐색/상호작용 |
| **웹** | 2개 | 웹 검색/콘텐츠 추출 |
| **비전** | 1개 | 이미지 분석 |
| **의사 결정** | 1개 | 작업 위임 (delegate_task) |
| **크론** | 1개 | 주기적 작업 예약 |
| **스킬** | 2개 | 스킬 관리/조회 |
| **세션** | 1개 | 과거 세션 검색 |
| **작업** | 1개 | TODO 리스트 관리 |

## 주요 도구 상세

### 📁 파일 도구

| 도구 | 설명 | 사용 예 |
|------|------|---------|
| `read_file` | 파일 내용 읽기 (자동 페이지네이션) | 코드 리뷰, 설정 확인 |
| `write_file` | 파일 생성/수정 (구문 검증 자동) | 코드 작성, 문서 편집 |
| `search_files` | 파일 내 검색 (ripgrep 기반) | 코드 찾기, 패턴 검색 |

> `cat`, `grep`, `find` 대신 이 도구들을 사용하세요. 더 효율적이고 구문 검증도 자동으로 수행합니다.

### 💻 터미널 도구

| 도구 | 설명 | 사용 예 |
|------|------|---------|
| `terminal` | 셸 명령어 실행 | 빌드, 설치, Git, 테스트 |
| `process` | 백그라운드 프로세스 관리 | 서버 실행, 장시간 작업 |

> **주의**: `cat`/`grep`/`find`/`sed`/`awk` 대신 파일 도구를 우선 사용하세요.

### 🌐 브라우저 도구

| 도구 | 설명 |
|------|------|
| `browser_navigate` | URL로 이동, 페이지 스냅샷 반환 |
| `browser_snapshot` | 페이지 접근성 트리 스냅샷 |
| `browser_click` | 요소 클릭 |
| `browser_type` | 입력 필드에 텍스트 입력 |
| `browser_scroll` | 페이지 스크롤 |
| `browser_vision` | 스크린샷 + 시각적 분석 |
| `browser_console` | 콘솔 로그 읽기/JS 평가 |

> 브라우저 도구는 동적 웹 페이지와의 상호작용이 필요할 때 사용합니다. 일반 텍스트나 API 엔드포인트는 웹 도구가 더 빠릅니다.

### 🔍 웹 도구

| 도구 | 설명 |
|------|------|
| `web_search` | 웹 검색 (키워드 기반) |
| `web_extract` | URL에서 콘텐츠 추출 |

### 👁️ 비전 도구

| 도구 | 설명 |
|------|------|
| `vision_analyze` | 이미지 URL/파일 분석, 캡차 해독 |

### 🔄 의사 결정 도구

| 도구 | 설명 |
|------|------|
| `delegate_task` | 하위 에이전트에 작업 위임 |

> 복잡한 작업을 여러 에이전트로 분할할 때 사용합니다.

### ⏰ 크론 도구

| 도구 | 설명 |
|------|------|
| `cronjob` | 주기적 작업 예약/관리 |

> "매일 아침 9시에 보고서 생성" 같은 반복 작업을 자동화합니다.

### 🎓 스킬 도구

| 도구 | 설명 |
|------|------|
| `skill_view` | 스킬 내용 로드 |
| `skill_manage` | 스킬 생성/수정/삭제 |
| `skills_list` | 사용 가능한 스킬 목록 |

## 도구 활성화/비활성화

### 설정 파일 (config.yaml)

```yaml
tools:
  enabled:
    - terminal
    - file
    - browser
    - web
    - vision
  disabled:
    - delegate_task  # 특정 도구 비활성화
```

### CLI 명령어

```bash
# 도구 목록 보기
hermes tools list

# 도구 활성화
hermes tools enable browser

# 도구 비활성화
hermes tools disable delegate_task
```

## 도구 사용 모범 사례

### 1. 파일 작업
```diff
- ❌ cat file.py | grep "TODO"
+ ✅ search_files "TODO" --file-glob "*.py"
```

### 2. 웹 콘텐츠
```diff
- ❌ browser로 API 문서 접속
+ ✅ web_extract로 API 문서 텍스트 추출
```

### 3. 긴 작업
```diff
- ❌ foreground에서 10분 대기
+ ✅ background=true + notify_on_complete=true
```

### 4. 이미지 분석
```diff
- ❌ 설명만 요청
+ ✅ vision_analyze(image_url)로 직접 분석
```

## 도구 선택 가이드

```
파일을 읽고 싶다면?
├─ 텍스트 파일 → read_file
├─ 이미지 파일 → vision_analyze
└─ 바이너리 파일 → terminal (file 명령어)

웹에서 정보를 얻고 싶다면?
├─ 정적 텍스트 → web_extract
├─ 동적 페이지 → browser_navigate
└─ 검색 결과 → web_search

작업을 실행하고 싶다면?
├─ 단순 명령어 → terminal (foreground)
├─ 긴 작업 → terminal (background)
├─ 반복 작업 → cronjob
└─ 복잡한 작업 → delegate_task
```

> **💡 Tip**: 올바른 도구를 선택하는 것이 효율적인 Hermes 사용의 첫걸음입니다. 각 도구의 특성을 이해하고 상황에 맞게 사용하세요.
