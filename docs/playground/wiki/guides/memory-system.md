## Memory 시스템이란?

Memory 시스템은 Hermes의 **영속적 데이터 저장 계층**입니다. 채팅 히스토리, 설정, 학습한 패턴 등 모든 데이터를 SQLite 데이터베이스에 저장합니다.

### Memory의 3가지 저장소

| 저장소 | 내용 | 용도 |
|--------|------|------|
| **세션 DB** | 대화 내용 전체 | 채팅 히스토리, 컨텍스트 유지 |
| **Knowledge DB** | 구조화된 지식 | 장기 기억, FTS5 검색 |
| **설정 파일** | config.yaml | 시스템 설정, 모델 라우팅 |

---

## Session DB 이해하기

모든 대화는 세션 DB에 저장됩니다. 다음 세션에서도 이전 대화를 검색할 수 있습니다.

### 자동 저장되는 데이터
- 사용자 메시지
- 에이전트 응답
- 도구 호출 및 결과
- JOB 상태 변경
- Knowledge 저장 및 검색

### 세션 검색 명령어

```
# 가장 최근 세션 보기
/sessions

# 특정 키워드로 세션 검색
/sessions search "API 배포"

# 특정 세션 열기
/sessions open JOB-1234
```

## Memory 저장 메커니즘

### 자동 저장

Hermes는 사용자와의 모든 상호작용을 **자동으로 저장**합니다. 별도의 저장 명령어가 필요 없습니다.

```
사용자: "안녕" 
→ 자동 저장: 세션 DB에 기록

사용자: "이 코드를 분석해줘"
→ 자동 저장: 요청 + 응답 + 도구 결과
```

### 명시적 저장

특정 정보를 영구 보관하려면 Knowledge를 사용합니다.

```
# Knowledge로 저장
knowledge add --tier T2 "이 프로젝트는 FastAPI + PostgreSQL 스택"

# 저장 확인
knowledge search "프로젝트 스택"
```

## Memory 유지보수

### 백업

정기적으로 Memory를 백업하는 것이 좋습니다:

```
# 전체 시스템 백업
hermes backup

# 특정 데이터베이스 백업
cp ~/.hermes/data/sessions.db ~/backups/sessions-2025-01-01.db
```

### 정리

오래된 데이터는 정리할 수 있습니다:

```
# 90일 이상 된 세션 정리
hermes cleanup --days 90

# 특정 Knowledge 삭제
knowledge delete <id>
```

## Memory와 Knowledge의 차이

| 구분 | Memory | Knowledge |
|------|--------|-----------|
| **저장 방식** | 자동 | 명시적 |
| **검색** | session_search() | FTS5 검색 |
| **구조** | 비구조화 (채팅 로그) | 구조화 (태그, 계층) |
| **수명** | 설정에 따라 자동 정리 | 명시적 삭제 전까지 영구 |
| **용도** | 컨텍스트 유지 | 장기 지식 보관 |

## 실전 팁

### 팁 1: 중요한 결정은 Knowledge에 저장하라
채팅 히스토리는 언제든 정리될 수 있습니다. 프로젝트 결정, API 키, 설정 값은 반드시 Knowledge에 저장하세요.

### 팁 2: session_search를 적극 활용하라
전에 했던 논의를 찾을 때 스크롤하지 말고 `session_search`를 사용하세요. FTS5가 빠르게 찾아줍니다.

### 팁 3: Memory는 무한하지 않다
Hermes는 컨텍스트 윈도우 내에서만 과거를 참조할 수 있습니다. Knowledge는 이보다 더 오래 기억합니다.

> **💡 Tip**: Memory 시스템은 Hermes의 기초입니다. 모든 상호작용이 기록되므로, 이전 작업을 안전하게 재개할 수 있습니다.
