Lecture C · Skill Structure
Skill 구조 — Pyramid Principle
결론 먼저: Skill은 정의(Definition) → 조건(Condition) → 실행(Steps) → 결과(Output)의 4단계 계층
📋 Definition Skill의 정체성 name, description
✅ Condition 실행 전제 조건 when: trigger, guard
⚙️ Steps 실행 절차 steps: [action, cmd, parse]
📤 Output 결과 포맷 format: json, text, raw
▸ 하위 단계 없으면 상위 단계 작동 불가 — bottom-up dependency
  • Definition: Skill의 고유식별 — name + description (catalog.json)
  • Condition: 언제 실행할지 — trigger pattern + guard check
  • Steps: 어떻게 실행할지 — action sequence (cmd/parse/handler)
  • Output: 결과 반환 형태 — structured → LLM 프롬프트로

Lecture C · Runtime Detail
Runtime Detail
PD L3 Detail ← C.4 Preview 응답
Step 1: Context Injection (M5)
  • 의도 분석: LLM이 요청의 true intent 파악
  • Memory 주입: FTS5 Top-K → 프롬프트에 삽입
  • Knowledge 참조: 관련 wiki 문서 자동 연결
  • 동시 주입: 3-Layer context를 한 번에 처리
Step 2: Routing
  • Skill 매칭: catalog.json trigger 패턴 비교
  • State 결정: 현재 workflow state 확인
  • Priority 계산: 여러 skill일 때 우선순위
Step 3: Compression (Token Budget)
// Runtime Pipeline — M5 Context Injection const context = { memory: memory.search(query, { topK: 5 }), knowledge: wiki.search(domain, { depth: 2 }), skills: catalog.match(intent), budget: { maxTokens: 4096, used: 0 } }; // Compression — budget 초과 시 truncation if (context.used > context.budget.maxTokens) { compress(context, { strategy: "summarize" }); }
