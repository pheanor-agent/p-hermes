Lecture A · SCQA — Situation
Stateless Problem
세션이 끝날 때 지워지는 모든 것 — 이것이 Agent의 근본 한계다
User 입력
→
LLM 추론
→
응답 생성
→
💨 모든 것 사라짐
  • 매 세션마다 처음부터 시작 — 컨텍스트 없음
  • 같은 질문을 매번 반복해야 함
  • Agent는 과거를 기억하지 못함
  • 사용자 경험: 매번 새로 설명하는 피로감
  • 핵심: Stateless = 학습 불가 = 진화 불가
💡 ChatGPT를 닫고 다시 열면 — 모든 것이 사라진다. 이것이 Stateless의 현실이다.
Lecture A · SCQA — Complication
4가지 실패 패턴
MECE 분류 — 모든 Stateless 실패는 이 4가지로 귀결된다
①
Context Loss
과거 대화 맥락이 사라짐
②
No Learning
과거에서 학습하지 못함
③
No Reusability
동일한 작업마다 재작성
④
No Consistency
일관된 결과 보장 불가
🔑 4가지 증상이지만 근본 원인은 동일: 상태를 저장하는 시스템이 없다
Lecture A · SCQA — Question
이게 진짜 문제인가?
Stateless vs Agent OS — 실제 영향력 차이가 결정적이다
✗ Stateless Agent
✗ 매 세션 컨텍스트 재설정
✗ 학습 누적 불가
✗ 같은 작업 매번 새로 작성
✗ 결과 일관성 없음
✗ 사용자는 매번 교육자 역할
✓ Agent OS
✓ Memory가 컨텍스트 보존
✓ Knowledge로 학습 누적
✓ Skills로 재사용 가능
✓ Workflow로 일관성 보장
✓ 사용자는 전략가 역할
이 차이를 해결하는 것이 Agent OS의 존재 이유다

왜 Memory인가?
기억 없이는 학습 불가 — 성장할 수 없는 Agent는 단순한 Chatbot일 뿐
🧠💨
Memory 없음
과거 대화 → 사라짐
학습 불가 → 성장 불가
매번처음부터 시작
🧠💾
Memory 있음
경험 보존 → 학습 가능
패턴 인식 → 개선
시간에 따른 진화
경험
→
저장 (Memory)
→
검색 (Recall)
→
학습 (Growth) ✓
  • Memory = Agent의 경험 축적 시스템
  • 과거 세션의 컨텍스트 보존 가능
  • Relevance-based 검색으로 정확한 정보 회수
  • 시간에 따른 지적 성장의 기반
Lecture A · Why Chain — 2
왜 Knowledge인가?
사실 ≠ 이해 — 데이터를 아는 것과 관계를 아는 것은 완전히 다른 차원
📄
사실만 있는 경우
데이터는 많지만 관계 모름
맥락 없는 정보 파편
도메인 이해 부족
🗂️
Knowledge가 있는 경우
정보 → 관계 → 의미
도메인 지식 구조화
문맥적 이해 가능
Raw Data
→
구조화
→
관계 추출
→
Knowledge ✓
  • Knowledge = 사실의 관계를 구조화
  • 도메인별 지식 체인 구성 (예: 프로그래밍, 마케팅)
  • 단순 정보 저장을 넘어 문맥적 이해
  • Memory의 다음 단계: 저장 → 이해

  • 동일 작업의 일관된 품질 보장
  • Knowledge의 다음 단계: 이해 → 실행
Lecture A · Why Chain — 4
왜 Workflow인가?
실행 ≠ 체계 — 개별 실행을 상태 머신으로 연결해야 완수 가능
🔧
실행만 있는 경우
개별 작업은 되지만 통합 안 됨
단계 간 연결 부재
전체 프로세스 깨짐
🔗
Workflow가 있는 경우
상태 머신으로 단계 관리
단계 간 의존성 자동 처리
체계적인 완수 보장
📋 Plan
↓
🔍 Research
↓
✍️ Execute
↓
✅ Review
↓
🎯 Done ✓
  • Workflow = 개별 실행 → 체계의 통합 계층
  • 상태 머신(state machine)으로 단계 관리
  • 단계 간 의존성 자동 처리
  • Skills의 다음 단계: 실행 → 완수
