Lecture C Layer 4: Workflow — Control 7/16
  • 9-Step State Machine — request에서 done까지 완전 제어
  • State-based — 단순 Step 모음이 아닌 상태 전이 모델
  • Gate-controlled — review(Step 4), approval(Step 5) 통과 체크
  • Rollback-safe — 실패 시 이전 State로 자동 복귀
  • Persistent — .workflow-state JSON에 전이 기록 저장

Lecture C 통합 포인트 — Layer 간 연결 8/16
Bottom-Up Context Flow
🔗 Workflow — 조율 · State Machine
⚡ Skills — 실행 · Subprocess
📚 Knowledge — 이해 · Wiki 구조
🧠 Memory — 경험 · FTS5+BM25
Top-Down Control Flow
  • Workflow가 design에서 Knowledge 참조
  • Workflow가 execution에서 Skills 호출
  • Workflow가 세션 히스토리로 Memory 활용
  • Skills 실행 시 3-Layer context 동시 주입
  • 실패 시 rollback으로 이전 State 복귀
하위에서 context가 올라가고, 상위에서 control이 내려감 — 양방향 흐름

Lecture C Gate 메커니즘 — 통과 규칙 12/16
.workflow-state
{"current": "execution", "history": ["request" → "approval"], "gate": {"review": "passed", "approval": "passed"}}
  • State 전이는 원자적 — 부분 실행 불가
  • .workflow-state JSON에 전이 기록 저장
  • review Gate: quality_score >= 0.7 통과
  • approval Gate: user: yes/no로 판단
  • 실패 시 정밀 rollback — 이전 State로 복귀
🚦 Review Gate (Step 4)
설계안 완성도 체크 — 실패 → design으로 rollback
🚦 Approval Gate (Step 5)
사용자 승인 — 거부 → investigation으로 rollback
↩️ Test Rollback (Step 7)
test 실패 → execution으로 정밀 복귀
Gate는 무작정 전진이 아닌 통제된 전진 — 품질을 보장하는 핵심 메커니즘
