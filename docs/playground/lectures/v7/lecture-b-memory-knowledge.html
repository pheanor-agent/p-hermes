BM25 Ranking

PD L3
BM25 점수 공식
# BM25 Scoring score(d, q) =   Σ IDF(qi) × TF(qi, d)     / (TF + k₁·(1-b+b·|d|/avgdl)) # 파라미터 k₁ = 1.2 # TF 상한 b = 0.75 # 길이 정규화
  • TF: 단어 출현 빈도
  • IDF: 희귀 단어 가중치 ↑
  • k₁: TF saturating 파라미터
  • b: 문서 길이 정규화 계수
실제 쿼리 포맷
-- 실제 FTS5 + BM25 쿼리 SELECT rowid, content, tags,   rank AS bm25_score FROM memories_fts WHERE memories_fts MATCH   "project AND structure" ORDER BY rank LIMIT 5;

②
최신 우선 Tie-break
점수 동률 시 최근 데이터 우선
③
K=5 반환
상위 5개만 LLM 프롬프트에 포함

Lecture B · Management

Memory Management

건강하게 성장한 Memory 시스템의 4가지 관리 전략

⚖️
Exposure-based Weighting 자주 검색된 Memory는 가중치 ↑ → 높은 순위
⏱️
TTL (Time-To-Live) 30일 미접근 Memory 자동 아카이브
🏷️
Tag-based Filtering 도메인/프로젝트/유저별 태그로 필터
🔄
Deduplication 유사 Memory 병합 + 중복 제거
  • Exposure: 검색 빈도가 높은 Memory는 BM25 점수에 가중치 추가
  • TTL: 30일 이상 미사용 Memory는 Archive로 이동 (삭제 아님)
  • Tag Filter: WHERE tags MATCH "python"로 검색 범위 축소
  • Dedup: 유사도 0.9 이상의 Memory 자동 병합

↓
← 사용자/세션 관련

📚 Knowledge 사용 "Python list comprehension" "HTTP 상태 코드 설명"

  • 조합: Memory(문맥) + Knowledge(지식) = 최적 응답

Skills: 아는 것 ≠ 실행하는 것

Feynman 원리 — 설명할 수 있다면 실행할 수 있어야 한다

✓ Skills가 있는 경우
✓ 절차화된 실행 모듈
✓ Skill 정의로 재사용
✓ 일관된 결과 품질
✓ 복잡한 작업도 체계적
💡 Feynman: "지식의 척도는 그 지식을 실행으로 옮길 수 있는 능력이다" — Skills Layer는 이 격차를 해결

Lecture B · Skill Structure

Skill 구조 — Pyramid Principle

결론 먼저: Skill은 정의(Definition) → 조건(Condition) → 실행(Steps) → 결과(Output)의 4단계 계층

📋 Definition Skill의 정체성 name, description
✅ Condition 실행 전제 조건 when: trigger, guard
⚙️ Steps 실행 절차 steps: [action, cmd, parse]
📤 Output 결과 포맷 format: json, text, raw
▸ 하위 단계 없으면 상위 단계 작동 불가 — bottom-up dependency
4단계 구조
  • Definition: Skill의 고유식별 — name + description (catalog.json)
  • Condition: 언제 실행할지 — trigger pattern + guard check
  • Steps: 어떻게 실행할지 — action sequence (cmd/parse/handler)
  • Output: 결과 반환 형태 — structured → LLM 프롬프트로
💡 Skills = Knowledge의 실행화 — 아는 것(Knowledge)이 아닌, 실행하는 구조(Skills)
