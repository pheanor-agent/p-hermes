Lecture B DIKW Pyramid 10/21
🧠Knowledge이해, 관계
📋Information맥락, 정리
🔢Data원시, 사실
🔑 Data는 '무엇', Information은 '어떻게', Knowledge는 '왜'
실제 예시
  • Data: HTTP 상태 코드 200
  • Information: 200 = 요청 성공
  • Knowledge: 왜 200이 필요한지 이해
Knowledge의 특징
  • 관계 기반 (문서 간 링크)
  • 도메인별 구조화
  • 재사용 가능한 단위
Knowledge는 관계 이해 — 단순 정보 나열이 아닌 구조화된 지식

Lecture B Skill 구조 — 4단계 계층 17/21
4단계 구조
  • Definition: Skill 정체성 (name + description)
  • Condition: 실행 전제 (trigger + guard)
  • Steps: 실행 절차 (cmd/parse/handler)
  • Output: 결과 포맷 (json/text/raw)
💡 하위 단계 없으면 상위 단계 작동 불가 — bottom-up dependency
catalog.json 예시
# lint skill 정의 { "name": "lint", "trigger": "lint|format", "condition": { "tool": "ruff|flake8" }, "steps": [ { "cmd": "ruff check" }, { "parse": "json" } ] }
Definition → Condition → Steps → Output, 4단계 bottom-up 구조
