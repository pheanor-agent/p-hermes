Maintenance 29 / 37

분리 ↔ 병합 — 적절한 크기

분리가 필요할 때

덩어리가 너무 큰 Skill

• 100줄이 넘는 긴 Skill
• 여러 작업을 한 파일에
→ 작은 Skill로 분리

병합이 필요할 때

너무 많은 작은 Skill

• 비슷한 Skill이 5개 넘게
• 서로 중복된 내용
→ 큰 Skill로 통합

"올바른 크기의 Skill이 가장 강력합니다"
너무 크면 복잡하고, 너무 작으면 찾기 힘듭니다.
한 Skill이 하나의 명확한 문제를 해결할 때 가장 효율적입니다

Deck C 34 / 37

4가지가 함께 작동할 때

Skill만 있으면

절차는 있지만 맥락 부족

• 표준 형식은 맞지만
• 사용자 스타일 반영 안 됨
• 프로젝트 정보 모름
• 다음 단계로 연결 안 됨

4가지 모두

맥락 + 절차 + 흐름

• 표준 형식 + 사용자 스타일
• 프로젝트 지식 활용
• 단계별 검증 + 다음으로
• 경험을 다음에 반영

"Skill은 핵심이지만, 혼자서는 부족합니다"
Memory + Knowledge + Workflow와 연결될 때
정말 강력한 Agent 시스템이 됩니다
