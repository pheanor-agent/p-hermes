🧠

두 종류의 기억

인간에도 단기 기억과 장기 기억이 있듯, Agent도 두 종류를 가집니다

💭

세션 메모리 — 현재 대화

지금 진행 중인 대화의 맥락. "아까 말씀드린 것처럼..." — 이 '아까'가 가능하게 합니다. 대화 중 실시간으로 업데이트됩니다.

📝

지속 메모리 — 장기 저장

세션이 끝나도 남는 기억. 사용자 선호도, 작업 패턴, 프로젝트 이력. 다음에 접속해도 계속 따라다닙니다.

💡 비유: 세션 메모리는 '지금 생각하는 것', 지속 메모리는 '일기에 적어둔 것'입니다.
탭을 클릭하면 메모리 유형이 전환됩니다

기억은 어떻게 저장될까요?

📋

4단계 기억 프로세스

1. 감지 → 대화에서 중요한 정보 포착
2. 분류 → 사용자 정보 / 환경 정보 구분
3. 저장 → SQLite 데이터베이스에 기록
4. 활용 → 다음 대화에서 자동 로딩

⚡

Skill은 하나의 파일에 모든 것

SKILL.md = Agent의 레시피 책. 하나의 파일에 절차, 주의사항, 검증 방법이 모두 포함됩니다

📋

트리거 — 언제 사용할지

어떤 상황에서 이 Skill이 발동될지 정의합니다. "보고서 작성 요청 시", "코드 리뷰 필요 시" 등 구체적인 조건을 명시합니다.

📝

단계 — 어떤 순서로 실행할지

1, 2, 3... 순서대로 실행하는 단계. 누가 봐도 이해할 수 있도록 명확하고 간결하게 작성합니다.

⚠️

주의사항 — 실패하지 않기 위해

과거에 발견된 버그, 주의할 점, 회피할 패턴. 실패 경험을 절차로 남깁니다.

💡 비유: Skill은 외식 대신 집에서 만드는 레시피입니다. 매번 같은 품질을 보장합니다.
