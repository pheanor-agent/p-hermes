# 지식 시스템 검색 및 활용

Hermes는 운영 기간 중 학습한 데이터(세션 이력, 작업 산출물, 외부 리퍼런스 등)를 가공하여 **계층적 지식 베이스(Knowledge Base)**에 저장합니다. 이 가이드는 해당 지식을 검색하고 작업에 활용하는 방법을 설명합니다.

## 📚 지식 시스템 구조

에이전트는 막연하게 "기억"하는 것이 아니라, 구조화된 파일 시스템에서 지식을 검색합니다.

1. **원본(Source)**: 세션 기록, JOB 산출물, 외부 뉴스, 리퍼런스 링크 등.
2. **가공 파이프라인**: 원본이 주기적으로(`wiki-process-filings.sh`) 가공되어 중복을 제거하고 핵심만 추출합니다.
3. **Wiki DB**: 가공된 데이터가 도메인(Domain)과 태그(Tag)로 분류되어 `~/.hermes/knowledge/wiki/`에 저장됩니다.
4. **FTS5 검색**: SQLite의 Full-Text Search를 사용하여 초고속으로 키워드를 탐색합니다.

---

## 🔍 지식 검색하기

### 1. 키워드 검색
에이전트는 사용자의 요청을 인식하면 자동으로 관련 지식을 검색합니다. 사용자가 직접 검색을 지시할 수도 있습니다.

> "이전에 '백업 시스템' 관련해서 논의된 내용을 찾아줘."

에이전트는 `session_search` 도구를 통해 과거 대화와 위키 데이터를 스캔하여 핵심 내용을 요약해 줍니다.

### 2. 도메인별 탐색
지식은 다음과 같은 주요 도메인으로 나뉩니다:
- `system`: 시스템 설정, 아키텍처 변경 이력.
- `dev`: 개발 규칙, 스킬, 테스트 메커니즘.
- `knowledge`: 지식 시스템 자체의 메타 데이터.
- `custom`: 사용자 맞춤형 워크플로우 및 리퍼런스.

## 🧠 지식 활용 시나리오

### 시나리오 A: 과거 결정 사항 확인
> "왜 심링크(Symlink)를 사용하지 않기로 했지?"

에이전트는 `knowledge/wiki/`에서 관련 결정(Decision) 레코드를 찾아, 그 당시의 논의 과정과 결론(예: LLM 탐색 효율 저하)을 보고합니다.

### 시나리오 B: 세션 이력 복원
> "작년 1월에 진행했던 GPU 대여 작업 내용을 다시 보여줘."

에이전트는 `session_search`를 통해 해당 시기의 JOB-XXXX 기록을 찾아, 설계서와 실행 결과를 다시 확인해 줍니다.

### 시나리오 C: 외부 지식 연계
> "Hermes와 함께 사용할 수 있는 MCP(Microsoft Copilot) 관련 정보가 있어?"

에이전트는 `knowledge/references/`의 외부 리퍼런스 인덱스를 확인하여 최신 정보를 검색합니다.

---

## ⚠️ 주의사항 (지식 한계)
- **추정 금지**: 에이전트는 없는 정보를 상상(Hallucinate)해서는 안 됩니다. 확실하지 않으면 "해당 정보가 기록되어 있지 않습니다"라고 솔직하게 보고합니다.
- **개인 데이터 보호**: 개인의 운영 통계(위키 페이지 수 등)와 운영자 구성(모델명)은 지식 시스템에서 추상화되어 저장됩니다.
