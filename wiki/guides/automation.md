# 자동화(Cron) 설정 가이드

Hermes는 수동 요청뿐만 아니라, 사용자가 정의한 규칙에 따라 **주기적으로 자동으로 작업을 수행**할 수 있습니다. 이를 Cron(크론) 시스템이라고 합니다.

## ⏰ 크론 시스템의 3계층 구조

Hermes의 크론 시스템은 신뢰성을 위해 3개의 계층으로 분리되어 작동합니다.

1. **Registry (`registry.yaml`)**: "무엇을 언제 실행할지"를 정의하는 설정 파일입니다. 모든 크론 작업의 SSOT(Single Source of Truth)입니다.
2. **Wrapper**: 크론 스케줄러가 실행하는 진입점 스크립트입니다. 환경 변수 로드 및 세션 관리를 담당합니다.
3. **Runner**: 실제 작업을 수행하는 에이전트(또는 스크립트)입니다.

---

## 🛠️ 크론 작업 만들기

### 방법 1: 채팅으로 요청하기 (가장 추천)
에이전트에게 직접 자동화 작업을 요청하면, 에이전트가 설계부터 등록까지 전 과정을 처리합니다.

**예시 요청:**
> "매주 월요일 오전 9시에 뉴스 리포트를 작성해서 디스코드로 보내주는 크론을 만들어줘."

**에이전트 동작:**
1. `prompt`: 뉴스 수집 및 요약 지침 정의.
2. `schedule`: `0 9 * * 1` (매주 월요일 9시) 설정.
3. `deliver`: `discord:1503941910459580450` 설정.
4. `registry.yaml`에 새 항목을 원자적으로(Atomic) 추가합니다.

### 방법 2: 수동 설정 파일 수정 (고급)
직접 `~/.hermes/cron/registry.yaml`을 편집하여 작업을 추가할 수 있습니다.

```yaml
- id: job-weekly-news
  name: "주간 뉴스 리포트"
  prompt: "최근 7일간의 기술 뉴스를 수집하고 요약해줘."
  schedule: "0 9 * * 1"
  deliver: "discord:1503941910459580450"
  enabled: true
```

## 🔄 크론 작업 관리

- **상태 확인**: "현재 실행 중인 크론 목록을 보여줘."
- **일시 중단**: "weekly-news 크론을 잠시 멈춰줘." (`pause`)
- **재개**: "weekly-news 크론을 다시 시작해줘." (`resume`)
- **삭제**: "weekly-news 크론을 영구적으로 제거해줘." (`remove`)

---

## 📥 전달(Delivery) 설정

크론 작업이 완료된 결과를 어디로 보내야 하는지 정의합니다.

| 타겟 | 값 예시 | 설명 |
|---|---|---|
| **현재 대화방** | `"origin"` | 크론을 생성한 채팅방으로 자동 회신 |
| **Discord 채널** | `"discord:1503941910459580450"` | 지정된 디스코드 서버/스레드로 전송 |
| **Telegram** | `"telegram:-1003938942705"` | 지정된 텔레그램 그룹으로 전송 |
| **로컬 파일** | `"local"` | `~/.hermes/cron/output/`에 파일로만 저장 |

## 💡 크론 활용 아이디어
- **정보 모니터링**: 블로그 RSS 피드, GitHub Issue, Polymarket 시장 동향 감시.
- **시스템 감시**: 디스크 용량, API 잔액, 프로세스 상태 확인 및 알림.
- **창작 보조**: 매일 아침 새로운 아이디어 브레인스토밍 기록.
- **데이터 백업**: 주기적인 코드베이스 또는 설정 파일 스냅샷.
