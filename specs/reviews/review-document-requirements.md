### 4.2 Wiki 섹션 구조 모순

공통 요구사항에 **「전제조건 → 설정 방법 → 단계별 실행 절차 → 검증 및 예외 처리」** 구조를 명시했지만, 실제 도메인별 구성은 「서론 → 내용 1~7 → 예외 처리」로 완전히 다른 패턴:

- A5 Wiki에는 서론이 있고 B2 Wiki에도 서론이 있음 → 공통 구조와 불일치
- 공통 구조의 '전제조건'은 어떤 도메인에도 명시적으로 존재하지 않음
- B2(아키텍처)는 설정/실행 절차가 없는 문서인데 동일 구조 강제는 부적절

**개선안**: 공통 구조를 두 레벨로 분리
- **가이드형** (A1-A6): 서론 → 전제조건 → 설정 → 실행 → 검증 → 예외
- **레퍼런스형** (B1-B2): 서론 → 개념 설명 → 구조 → 구성 요소 → 관련 링크

## 7. 일관성

### 7.1 유지되는 일관성 (좋은 점)

- ✅ Slides 구조: 후크 → 문제 → 해법 → 3원칙 → 단계별 → 요약 (8도메인 모두 동일)
- ✅ Blog 구조: 서론 → 문제 분석 → 핵심 메커니즘 → 전망 (일관적)
- ✅ 파일명 패턴: `{주제}.md`(Wiki), `{서사적-타이틀}.md`(Blog), `{주제}.html`(Slides)
- ✅ Mermaid JS: 모든 매체에서 통일된 다이어그램 도구

### 7.2 깨진 일관성 (수정 필요)

| 항목 | 문제 | 영향 |
|------|------|------|
| **YAML 프런트매터** | "Wiki/Blog 최상단"이라고 명시, Slides는 포함 안됨 | Slides도 메타데이터 필요할 수 있음, 명시적 배제 또는 포함 결정 필요 |
| **부정대조 패턴 금지** | Wiki/Blog에 명시, Slides에 명시 안됨 | Slides에도 동일한 규칙이 적용되는지 명확화 필요 |
| **시각화 도구** | Wiki 공통에서는 "시퀀스 다이어그램 또는 흐름도", 개별에서는 "Mermaid" | Wiki 공통을 "Mermaid JS"로 통일 |
| **Slides 제목 언어** | A1-A6: 영어 4건, 한국어 0건 (도메인명만 사용) | Blog는 한국어 서사적 제목, Slides는 과도하게 단순 |
| **"시각화" 표기** | Slides 영역에서 4건 모두 "시작화" 오타 | Slides Slides 섹션에서만 집중 발생 — Slides 파트 작성 시 systematic error |
| **Blog 파일명 패턴** | A2, A4, A6, B2는 `-design.md` 접미사, A1/A5는 `why-`, `cron-` 등 서사적 | Blog 파일명 패턴 두 종류 병존, 통일 권장 |
