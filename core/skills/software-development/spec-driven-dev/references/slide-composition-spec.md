# 슬라이드 구성 규격 설계 (JOB-1511, 1517)

## 문제 정의

사용자 질문: "사양서 내용만 가지고 현재 슬라이드 내용을 똑같이 만들 수 있는 수준의 디테일이 포함되있어?"

### SPEC-B6(슬라이드 버전 관리)의 한계

- 버전 관리 워크플로우만 정의
- "어떤 슬라이드가 필요한지" 정의 안 됨
- "슬라이드별 제목/라벨/포함 요소" 정의 안 됨
- "본문 vs 팝업 분리 규칙" 정의 안 됨

### SPEC-B7(슬라이드 구성 규격)의 필요성

| Spec ID | 목적 | 내용 |
|---------|------|------|
| SPEC-B6 | How | 버전 관리 워크플로우 |
| SPEC-B7 | What | 슬라이드 구성 구조 + 작성 규칙 |

## 슬라이드 구성 규격 템플릿

### 슬라이드 목록 정의
```yaml
슬라이드 목록:
  - 번호: 1
    제목: "AI 에게 기억을 선물하다"
    라벨: "Cover"
    목적: "Cover"
```

### 슬라이드별 내용 구조
```
- 라벨: 에이전트 메모리 시스템
- 제목: "AI 에게 기억을 선물하다"
- 부제목: "챗봇에서 파일 기반 지식 시스템으로"
- 설명: "사용자의 입력을 체계적으로 저장하고 활용하는 방법"
- 메타: 날짜, 발표자
```

### 본문 vs 팝업 분리 규칙
| 위치 | 내용 | 예시 |
|------|------|------|
| 본문 | 컨셉과 개략적 알고리즘 | "파일을 정리하는 3 단계" |
| 팝업 | 세부 적용 기술 | "Karpathy 3 층 구조, append-only, metadata.json" |

### 다이어그램 사용 규칙
| 동작 유형 | 다이어그램 사용 | 예시 |
|-----------|----------------|------|
| 순차적 동작 | 사용 | 수집 → 분류 → 점수 매기기 → 노출 |
| 분류적 동작 | 사용 | T1/T2/T3 분류 |
| 단순 설명 | 사용하지 않음 | 주제 분류 소개 |

### 언어 규칙
- 기술 용어: 영어 사용 (RAG, LLM, API)
- 나머지: 한글 표기
- 중국어: 사용 금지

### 팁 사용 규칙
- 본문과 중복 내용: 금지
- 뻔한 내용: 금지
- 추가 통찰 또는 실제 사례: 권장

## ⚠️ 중복 방지 규칙 (JOB-1517 학습 — 사용자 직접 지적)

**사용자 지적**: "같은 내용이 본문, 다이어그램, 팝업에 여러번 반복되는 문제는 안생기게 사양서가 작성되있어?"

### 문제

SPEC-B7 은 "중복 금지"만 선언하고, **어떻게 검증할지** 정의하지 않음. 실제 슬라이드에서 중복 발생 (슬라이드 5, 6 동일 팝업 내용).

### 슬라이드 내 중복 방지

| 위치 | 허용 내용 | 금지 내용 |
|------|-----------|-----------|
| 본문 | 개념/컨셉/개략적 알고리즘 | 다이어그램 요약, 팝업 내용 중복 |
| 다이어그램 | 흐름/구조/순서 | 본문 내용 반복, 텍스트-heavy |
| 팝업 | 기술적 세부사항/구체적 명령어 | 본문 내용 반복, 다이어그램 설명 |

### 슬라이드 간 중복 방지

| 중복 유형 | 규칙 | 예외 |
|-----------|------|------|
| 본문 ↔ 본문 | 금지 | — |
| 다이어그램 ↔ 다이어그램 | 금지 | — |
| 팝업 ↔ 팝업 | **허용** (공통 개념) | — |
| 본문 ↔ 다이어그램 | 금지 | — |
| 본문 ↔ 팝업 | 금지 | — |
| 다이어그램 ↔ 팝업 | 금지 | — |

### 검증 기준

- **자동 검증**: 텍스트 유사도 80% 이상 → 경고
- **수동 검증**: 작성자/리뷰어 확인
- **도구**: Python difflib, cosine similarity (선택)

### 올바른 Spec 작성 패턴

```markdown
### 슬라이드 5: 시스템 개요
- 다이어그램: 원본 데이터 → AI 가 정리 → 구조화된 지식
- 통계: 3 단계 파일 정리, 10 개 주제 분류
- 팝업: Karpathy 3 층 구조 (기술적 세부사항만)

### 슬라이드 6: 파일 정리 3 단계
- 다이어그램: 1 단계 원본 → 2 단계 정리 → 3 단계 규칙
- 카드 3 개: 원본/정리/규칙 (본문은 개념만)
- 팝업: Karpathy 3 층 구조 (공통 개념 → 허용)
```

**징후**: 사용자가 "같은 내용이 여러번 반복되는 문제가 안생기나?", "중복 방지 규칙이 있어?" 지적 → 중복 방지 규칙 누락
