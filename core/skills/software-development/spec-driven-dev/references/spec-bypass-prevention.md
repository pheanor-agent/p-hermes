# Spec 우회 방지 및 콘텐츠 분리 패턴 (JOB-1519/1520)

## 문제 발견 (사용자 직접 지적)

**사용자 질문**: "사양서 내용을 기반으로 작성한다고 가정했을 때 같은 내용이 본문, 다이어그램, 팝업에 여러번 반복되는 문제는 안생기게 사양서가 작성되있어?"

**분석 결과**: 
- ✅ 피드백 반영: 대부분 Spec 에 반영됨
- ❌ Spec 우회 가능성 HIGH: 슬라이드 내용이 HTML 에 하드코딩 → 직접 수정 가능
- ❌ 누락 항목: 슬라이드 내용 Spec 파일, 코드 생성 파이프라인

## Spec 우회 방지 패턴

### 문제 상황
```
변경 요청 → docs/slides.html 직접 수정 → Git 푸시
              ↑ Spec 무시 (우회 발생)
```

### 해결 방안

#### 1. 코드 생성 파이프라인 도입
```
specs/content/*.md → scripts/generate-slides.py → docs/slides.html
```

#### 2. 직접 수정 금지
- `docs/slides.html` 은 자동 생성 파일
- 파일 헤더에 "DO NOT EDIT - Generated by scripts/generate-slides.py" 명시
- pre-commit hook 에서 직접 수정 감지 (선택)

#### 3. 검증 시스템 강화
```bash
# Spec ↔ HTML 일관성 검증
python scripts/validate-slides.py docs/slides.html

# 중복 항목 확인
python scripts/validate-slides.py docs/slides.html --threshold 0.75
```

## 콘텐츠 분리 규칙

### 슬라이드 내 분리
| 섹션 | 역할 | 예시 |
|------|------|------|
| 본문 | 개념/컨셉/개략적 알고리즘 | "파일을 정리하는 3 단계" |
| 다이어그램 | 흐름/구조/순서 | 수집 → 분류 → 점수 매기기 → 노출 |
| 팝업 | 기술적 세부사항/구체적 명령어 | "Karpathy 3 층 구조, append-only" |

### 중복 방지 규칙

#### 슬라이드 내
- 본문 ↔ 다이어그램: **금지**
- 본문 ↔ 팝업: **금지**
- 다이어그램 ↔ 팝업: **금지**

#### 슬라이드 간
- 본문 ↔ 본문: **금지**
- 다이어그램 ↔ 다이어그램: **금지**
- 팝업 ↔ 팝업: **허용** (공통 개념)

### 검증 기준
- **자동**: 텍스트 유사도 80% 이상 → 경고
- **수동**: 작성자/리뷰어 확인
- **도구**: Python difflib.SequenceMatcher

**검증 스크립트 패턴 **(scripts/validate-specs.sh)
```bash
#!/bin/bash
SPEC_DIR="specs/content"
EXPECTED_COUNT=11

# 1. 파일 수 검증
FILE_COUNT=$(ls ${SPEC_DIR}/*.md | wc -l)
if [ ${FILE_COUNT} -ne ${EXPECTED_COUNT} ]; then
    echo "❌ FAIL: 파일 수 불일치"
    exit 1
fi

# 2. 중복 파일 검증
python3 scripts/validate-slides.py docs/slides.html

# 3. Traceability 검증 (SPEC-B7 버전 정보 확인)
SPEC_VERSION=$(grep "version:" specs/active/components/slide-structure.md | head -1 | awk '{print $2}')
```

**빌드 스크립트 패턴 **(scripts/build-artifact.sh)
```bash
#!/bin/bash
# 1. Spec 버전 읽기 → 2. 검증 실행 → 3. HTML 생성 → 4. 버전 파일 저장 → 5. CHANGELOG 업데이트
bash scripts/validate-specs.sh || exit 1
```
