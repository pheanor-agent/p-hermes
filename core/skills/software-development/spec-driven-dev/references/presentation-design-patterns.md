# 프레젠테이션 설계 패턴 (JOB-1503 학습)

## 아이스 브레이킹 → 연결 → 본론 흐름

**사용자 선호**: 기술적 내용 바로 시작 금지. 비전공자도 접근 가능한 구조.

```
1. 아이스 브레이킹 (상황 설정)
   → "신입 사원,业务知识 어떻게 관리할까?"
   → 구체적 시나리오 제시
   
2. 연결 설명 (두 상황의 공통점)
   → "신입 사원과 AI의 공통점"
   → "두 경우 모두 '지식 관리 시스템'이 핵심입니다"
   
3. AI 기술 3층위 (기술적 배경)
   → 모델 학습 → 에이전트 메모리 → RAG
   → "오늘 주제" 명시적 표시
   
4. 본론 전환
   → "에이전트 메모리, 왜 중요한가요?"
   → "지식 시스템은 AI의 '외부 뇌'입니다"
```

## 기술적 내용 선택적 노출

**사용자 선호**: 핵심 내용은 필수, 기술적 세부사항은 선택적.

- **팝업/모달 패턴**: "더보기" 버튼 → 기술적 내용 팝업
- **Escape 닫힘**: Escape 키 또는 오버레이 클릭 시 닫힘
- **본문/상세 분리**: 본문에는 개념 설명만, 상세에는 수치/코드 포함

## 슬라이드 구성 원칙

| 항목 | 값 |
|------|-----|
| 슬라이드당 텍스트 | 200-400자 이하 |
| 화면 비율 | 16:9 (1920x1080) |
| 스크롤 | 완전 제거 (`overflow: hidden` + `100vh`) |
| 도입부 | 아이스 브레이킹 → 연결 → 본론 |
| 기술 내용 | 선택적 팝업 |
| 구체적 수치 | 개념 중심 (3,651 페이지 등 내부 수치 제거) |
