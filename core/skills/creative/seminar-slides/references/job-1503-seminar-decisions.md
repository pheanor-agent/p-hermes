# JOB-1503~1511 세미나 슬라이드 학습 사항

## 슬라이드 구성 규격 필요성 (JOB-1503/1511)

SPEC-B6(버전 관리)만으로는 슬라이드 내용을 재현할 수 없음. 구조적 규격(SPEC-B7) 필요:
- 슬라이드별 라벨 정의
- 제목 구조
- 포함 요소 (본문/다이어그램/팝업)
- 작성 가이드라인

## 본문 vs 팝업 분리 (JOB-1511)

- 본문: 컨셉과 개략적 알고리즘
- 팝업: 세부 적용 기술 (Karpathy 3층 구조, append-only, metadata.json 등)
- 본문의 목적: 비기술적 청중도 이해할 수 있는 수준

## 다이어그램 사용 규칙 (JOB-1511)

- 순차적 동작: 다이어그램 사용 (예: 수집 → 분류 → 점수 매기기 → 노출)
- 분류적 동작: 다이어그램 사용 (예: T1/T2/T3 분류)
- 단순 설명: 다이어그램 미사용 (예: 주제 분류 소개)
- 다이어그램은 본문에 직접 포함, 팝업이 아님

## 언어 규칙 (JOB-1506/1510)

- 기술 용어: 영어 사용 (RAG, LLM, API, GitHub Pages)
- 나머지: 한글 표기
- 중국어: 사용 금지 (원천 차단)
- 표기 일관성: 같은 개념은 같은 용어로 통일

## 팁 사용 규칙 (JOB-1510)

- 금지: 본문과 중복되는 내용
- 금지: 뻔하거나 자명인 내용
- 권장: 추가 통찰, 실제 사례, 비유적 설명

## RAG 정의 변경 (JOB-1510)

- 기존: "실시간 외부 검색"
- 변경: "외부 지식을 저장해두고 상황에 따라 바로 참조하는 구조"
- 이유: 기술적 오해 방지 (실시간 검색이 아닌, 미리 준비된 지식 참조)

## 슬라이드 구조 (JOB-1503/1511)

11개 슬라이드 구성:
1. Cover
2. 문제 제기
3. 신입사원 예시 (AI 내용 없음)
4. AI 의 3 층 기억 + 연결
5. 시스템 개요
6. 파일 정리 3 단계
7. 자동화 과정
8. 점수 → 중요도 연결
9. 주제 분류
10. 지식 찾는 과정
11. Thank You
